## 项目亮点

- **50 讲完整覆盖** — 从绪论到 I/O 子系统，四大部分一讲不落
- **AI 结构化笔记** — 每讲按固定五节格式（主题 → 知识点 → 术语 → 对比 → 总结）提炼
- **中英双语术语** — 关键术语首次出现均标注中英文对照
- **干净文本数据** — 去除口语冗余，可直接用于 RAG / 提示词工程

## 如何使用

| 场景 | 方式 |
|------|------|
| **在线浏览** | 直接在 GitHub 上点击 `notes/` 目录，浏览渲染好的 Markdown 笔记 |
| **离线学习** | 克隆仓库后用 [Obsidian](https://obsidian.md)、[Typora](https://typora.io) 等工具阅读，支持双链和搜索 |
| **搜索复习** | 利用 GitHub 仓库内搜索（快捷键 `t`）或本地全文检索，快速定位知识点 |
| **AI 知识库** | 笔记为结构化干净文本，可直接用于 RAG 管线、提示词工程或语料库构建 |

## 笔记格式说明

每个笔记文件遵循统一的五节结构，便于快速定位和横向对比：

```markdown
# 章节名（副题）

## 本讲主题          ← 1-2 句摘要
## 核心知识点        ← 4-7 个加粗一级条目 + 二级细节
## 关键术语          ← 5-8 条中英文对照表格
## 重要对比/关系     ← 条件节，有横向比较时出现
## 要点总结          ← 固定 5 条，关键词加粗
```
