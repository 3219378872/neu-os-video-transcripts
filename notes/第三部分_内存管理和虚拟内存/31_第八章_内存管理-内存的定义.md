# 第八章 内存管理——内存的定义

## 本讲主题

介绍内存的基本概念、地址绑定过程及逻辑地址到物理地址的转换机制。

## 核心知识点

- **内存的定义**
  - 内存是硬件实现的大字节数组，每个元素有唯一的绝对地址
  - 所有程序执行前必须加载到内存中
  - CPU 唯一能直接访问的大容量存储；二级存储（Secondary Storage）用于持久化备份
- **地址保护机制**
  - 每个进程分配基地址寄存器（Base Register）和界限地址寄存器（Limit Register）
  - 访问地址须满足：基地址 ≤ 访问地址 < 基地址 + 界限地址
  - 违规访问将陷入操作系统并返回错误
- **程序加载流程**
  - 编译（Compile）：源代码 → OBJ 文件（机器码）
  - 链接（Link）：多个 OBJ + 库函数 → 可载入单元
  - 加载（Load）：可载入单元 → 内存中运行
- **地址形式的变化**
  - 源代码中为符号型变量（如 `int a = 100`）
  - 编译链接后变为可重定位地址（相对程序起始的偏移量）
  - 加载运行后转为绝对地址（物理地址）
- **地址绑定时机**
  - 编译时绑定：需预知程序在内存中的位置，几乎不可行
  - 加载时绑定：加载过程中将可重定位地址转为绝对地址
  - 运行时绑定（默认采用）：变量以可重定位地址形式保存在内存，执行时动态转换
- **逻辑地址与物理地址**
  - 逻辑地址（虚地址）：CPU 产生的地址，即可重定位地址
  - 编译时/加载时绑定 → 逻辑地址 = 物理地址
  - 运行时绑定 → 逻辑地址 ≠ 物理地址
- **MMU 地址转换**
  - MMU（Memory Management Unit）内含重定位寄存器（Relocation Register）
  - 物理地址 = 逻辑地址 + 重定位寄存器值
  - 示例：逻辑地址 346 + 重定位寄存器 14000 = 物理地址 14346

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 内存 | Main Memory | 硬件实现的字节数组，CPU 直接访问的存储 |
| 二级存储 | Secondary Storage | 用于持久化备份的存储设备（如磁盘） |
| 可重定位地址 | Relocatable Address | 相对于程序起始位置的偏移量 |
| 地址绑定 | Address Binding | 将可重定位地址映射为绝对地址的过程 |
| 逻辑地址 | Logical Address | CPU 产生的地址，也称虚地址 |
| 物理地址 | Physical Address | 内存单元的实际绝对地址 |
| 内存管理单元 | MMU | 完成逻辑地址到物理地址转换的硬件 |

## 要点总结

1. 内存是**大字节数组**，每个元素有唯一绝对地址，程序运行前必须加载到内存
2. **基地址寄存器和界限地址寄存器**共同实现进程间的内存保护
3. 程序从源代码到运行经历**编译→链接→加载**三个阶段，地址形式逐步转换
4. 地址绑定默认发生在**运行时**，变量以可重定位地址形式保存在内存中
5. **MMU** 通过重定位寄存器将逻辑地址加上基地址，完成到物理地址的转换
