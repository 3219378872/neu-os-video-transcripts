# 第八章 内存管理——分段

## 本讲主题

介绍分段内存分配方案的原理、段表结构及与分页的对比。

## 核心知识点

- **分段基本思想**
  - 从用户角度看，程序是一系列段的集合（主函数段、子函数段、栈、符号表等）
  - 不同段大小不一，按功能划分
  - 以非连续方式为各段分配物理内存
- **逻辑地址结构**
  - 由段号（s）和段偏移量（d）两部分组成
  - 段表（Segment Table）将二维逻辑地址转换为一维物理地址
- **段表结构**
  - 每项包含两个字段：基地址（Base）和界限（Limit）
  - Base：该段在物理内存中的起始地址
  - Limit：该段的长度
  - 段表基地址寄存器（STBR）：保存段表在内存中的起始位置
  - 段表长度寄存器（STLR）：保存段表的项数
- **地址转换过程**
  - 由段号 s 查段表，获取 Base 和 Limit
  - 检查偏移量 d < Limit（合法性验证）
  - 物理地址 = Base + d
  - 示例：段0 基地址 1400、长度 1000 → 占用物理内存 1400~2400
- **分段的优势**
  - 共享代码：不同进程可共享代码段，物理内存中只需一个副本
  - 段级保护：可按段的性质设置保护措施（如代码段禁止修改）
  - 段页式（Segmentation with Paging）：在分段基础上进一步分页，接近实际操作系统方案

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 分段 | Segmentation | 按功能将程序划分为不等大段的内存分配方式 |
| 段表 | Segment Table | 记录每段基地址和长度的数据结构 |
| 段号 | Segment Number | 逻辑地址中标识所属段的编号 |
| 基地址 | Base | 段在物理内存中的起始地址 |
| 界限 | Limit | 段的长度 |
| 段页式 | Segmentation with Paging | 分段与分页结合的内存管理方案 |

## 重要对比/关系

| 特性 | 分页 | 分段 |
|------|------|------|
| 划分依据 | 固定大小的帧/页 | 按功能划分的不等大段 |
| 用户视角 | 用户不感知 | 符合用户对程序的逻辑认知 |
| 地址结构 | 页号 + 页偏移 | 段号 + 段偏移 |
| 碎片类型 | 内部碎片 | 外部碎片 |
| 映射数据结构 | 页表（帧号） | 段表（基地址 + 界限） |

## 要点总结

1. 分段从**用户视角**出发，将程序按功能划分为主函数、子函数、栈等不等大的段
2. 段表每项记录**基地址和界限**两个字段，用于地址转换和合法性验证
3. 地址转换：检查偏移量 < Limit 后，**物理地址 = Base + d**
4. 分段支持**代码共享**和**段级保护**（如禁止修改代码段）
5. **段页式**结合分段与分页的优点，是接近实际操作系统的内存管理方案
