# 第八章 内存管理——分页

## 本讲主题

介绍分页内存分配方案的原理、页表实现方式及 TLB 加速机制。

## 核心知识点

- **分页基本原理**
  - 物理内存分为等大的块，称为帧（Frame）
  - 进程逻辑地址空间也分为等大的块，称为页（Page）
  - 内存分配以帧为单位，将非连续的帧映射到页上
  - 页表（Page Table）记录每个页对应的帧号
- **逻辑地址结构**
  - 由页号（p）和页偏移量（d）两部分组成
  - 逻辑地址空间 m 位，页大小 n 位 → 页数 = 2^(m-n)
  - 物理地址 = 页表中查得的帧基地址 + 偏移量 d
- **分页的碎片特性**
  - 无外部碎片（按帧分配，不会产生进程间空隙）
  - 存在内部碎片（最后一页可能未填满）
  - 页大小典型值：4KB 或 8KB
- **页表的实现**
  - 理想方案：寄存器实现（速度快但数量不足）
  - 实际方案：页表保存在内存，用页表基地址寄存器（PTBR）和页表长度寄存器（PTLR）定位
  - 内存中保存页表的问题：每次数据访问需两次内存访问（一次查页表 + 一次读数据）
- **TLB 加速机制**
  - TLB（Translation Look-aside Buffer）：关联内存，速度介于寄存器和内存之间
  - 保存页表的部分常用条目（页号-帧号对）
  - TLB 命中 → 直接获取帧号，一次内存访问
  - TLB 未命中 → 访问内存中的完整页表，两次内存访问
- **有效/无效位（Protection Bit）**
  - 页表每项附加一个位，标识该页是否属于进程的有效逻辑地址空间
  - 有效（v）：该页存在于进程地址空间中
  - 无效（i）：该页不属于进程，访问将产生错误
- **共享页（Shared Pages）**
  - 多个进程可共享可重入代码（Reentrant Code），如文本编辑器
  - 共享进程的页表中相关页指向相同的物理帧
  - 私有数据各自独立分配

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 帧 | Frame | 物理内存中等大的块 |
| 页 | Page | 进程逻辑地址空间中等大的块 |
| 页表 | Page Table | 记录页到帧映射关系的数据结构 |
| TLB | Translation Look-aside Buffer | 缓存页表常用条目的高速关联内存 |
| 有效/无效位 | Valid/Invalid Bit | 标识页表项是否有效的保护位 |
| 可重入代码 | Reentrant Code | 运行中不被修改、可被多进程共享的代码 |

## 要点总结

1. 分页将物理内存分为**帧**、逻辑空间分为**页**，通过页表建立非连续映射
2. 分页**无外部碎片**但存在内部碎片，最后一页可能未填满
3. 页表保存在内存会导致**两次内存访问**，使用 TLB 缓存可降至一次
4. 页表的**有效/无效位**标识页是否属于进程合法地址空间
5. 多个进程可通过页表共享**可重入代码**，私有数据各自独立
