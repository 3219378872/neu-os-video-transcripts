# 第九章 虚拟内存——请求分页

## 本讲主题

介绍按需调页的工作机制、有效/无效位的作用及页错误的处理流程。

## 核心知识点

- **按需调页基本原理**
  - 进程采用分页管理方案，所有页初始在后备存储中
  - 运行时需要哪个页就加载哪个页，不需要时写回后备存储
  - 优点：更少的内存占用、更少的 I/O 操作
- **有效/无效位的双重作用**
  - 分页中：标识页是否属于进程的合法逻辑地址空间
  - 虚拟内存中：标识页是否已加载到物理内存
  - Valid（1）：页在内存中，可直接访问
  - Invalid（0）：页在后备存储中，或为非法访问
- **惰性交换（Lazy Swapper）**
  - 进程运行前所有页都在后备存储中，内存中没有任何页
  - 需要一个页就从后备存储调入一个，逐页按需加载
- **页错误（Page Fault）**
  - 访问的页不在内存中（有效/无效位为 Invalid）时产生页错误
  - 由操作系统判断是页错误还是非法访问
  - 示例：8 页进程，页表中 0 号页(帧4, Valid)、1 号页(Invalid)、2 号页(帧6, Valid)
  - 访问 0 号页直接从内存读取；访问 1 号页触发页错误，需从后备存储调入
- **页错误处理流程**
  - 步骤1：访问页表，发现有效/无效位为 Invalid
  - 步骤2：陷入操作系统（Trap），判断为页错误
  - 步骤3：在后备存储中找到对应页的位置
  - 步骤4：找到空闲帧，将页从后备存储加载到内存
  - 步骤5：更新页表（填写帧号，有效/无效位设为 Valid）
  - 步骤6：重启进程，从中断处继续执行

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 按需调页 | Demand Paging | 运行时按需加载所需页到内存的技术 |
| 有效/无效位 | Valid/Invalid Bit | 标识页是否在内存中的页表附加位 |
| 惰性交换 | Lazy Swapper | 初始不加载任何页，需要时才调入的策略 |
| 页错误 | Page Fault | 访问的页不在内存中时产生的中断 |
| 陷阱 | Trap | 页错误发生时陷入操作系统的机制 |

## 要点总结

1. 按需调页只加载**当前需要的页**，减少内存占用和 I/O 操作
2. **有效/无效位**在虚拟内存中用于标识页是否已在内存中
3. **惰性交换**策略下进程初始时所有页都在后备存储中
4. 访问不在内存中的页会产生**页错误**，需陷入操作系统处理
5. 页错误处理的核心流程：**找页→找帧→加载→更新页表→重启进程**
