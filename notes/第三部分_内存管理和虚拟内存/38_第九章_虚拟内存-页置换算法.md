# 第九章 虚拟内存——页置换算法

## 本讲主题

介绍 FIFO、OPT 和 LRU 三种页置换算法的原理、执行过程及 Belady 异常。

## 核心知识点

- **页引用串的生成**
  - 从内存引用地址中提取页号，连续访问同一页的合并为一次
  - 示例：页大小 100，地址 0100→页1，0432→页4，0612→页6
  - 正常情况下帧数越多，页错误数越少
- **FIFO 页置换算法**
  - 记录每个页进入内存的时间，选择最早进入内存的页作为牺牲帧
  - 注意：是最早**进入**内存的时间，不是最近访问时间
  - 示例（引用串 7,0,1,2,0,3,0,4，3帧）：7→0→1→换7为2→0命中→换0为3→...
  - 优点：实现简单
  - 缺点：重度使用的页可能被反复换入换出
- **Belady 异常**
  - 分配帧数增加，页错误数反而增加的反常现象
  - 示例：引用串 1,2,3,4,1,2,5,1,2,3,4,5，3帧产生 9 次页错误，4帧产生 10 次页错误
  - FIFO 算法可能出现此异常
- **OPT 最优页置换算法**
  - 选择**未来最长时间不会被使用**的页作为牺牲帧（向前看/将来时）
  - 能产生最少的页错误数
  - 缺点：需预知未来的页引用序列，实际无法实现，仅作为理论最优基准
- **LRU 页置换算法**
  - 选择**过去最长时间未被使用**的页作为牺牲帧（向后看/完成时）
  - 记录每个页的上次访问时间，选择最久未访问的页换出
  - 与 FIFO 的区别：FIFO 看进入时间，LRU 看最近访问时间

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| FIFO 算法 | First-In First-Out | 置换最早进入内存的页 |
| OPT 算法 | Optimal Algorithm | 置换未来最长时间不会使用的页 |
| LRU 算法 | Least Recently Used | 置换过去最长时间未被使用的页 |
| Belady 异常 | Belady's Anomaly | 帧数增加但页错误数反增的现象 |
| 页引用串 | Reference String | 进程访问的页号序列 |

## 重要对比/关系

| 算法 | 选择牺牲帧的依据 | 时间方向 | 可实现性 |
|------|------------------|----------|----------|
| FIFO | 最早进入内存的页 | 看过去（进入时间） | 可实现，简单 |
| OPT | 未来最久不使用的页 | 看未来 | 不可实现，理论最优 |
| LRU | 过去最久未访问的页 | 看过去（访问时间） | 可实现，效果好 |

## 要点总结

1. **FIFO** 按页进入内存的先后顺序置换，实现简单但可能出现 Belady 异常
2. **Belady 异常**指帧数增加反而页错误增多，源于重度使用的页被反复调入调出
3. **OPT** 置换未来最久不用的页，理论最优但无法预知未来，不可实际使用
4. **LRU** 置换过去最久未访问的页，是 OPT 的实际近似，效果较好
5. FIFO 看**进入时间**、LRU 看**访问时间**，两者极易混淆需注意区分
