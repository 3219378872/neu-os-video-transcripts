# 第二章 操作系统的历史（下）

## 本讲主题

从 GNU 项目到 Linux 诞生：自由软件运动、GPL 许可证与现代开源操作系统的起源。

## 核心知识点

- **Richard Stallman 与 GNU 项目**
  - Stallman：MIT 人工智能实验室的早期黑客（极客/Geek）
  - 背景：实验室更换设备后 UNIX 已被收回版权，无操作系统可用
  - **GNU = GNU's Not UNIX**：递归缩写
  - GNU 项目开发历程：
    - 第一个项目：自己写 UNIX 内核 → **失败**（难度超出想象）
    - 第二个项目：开发 UNIX 编译器 → **失败**
    - 第三个项目：开发文本编辑器 **Emacs** → **成功**
    - 用 Emacs 销售收入反哺前两个项目
  - GNU 项目的重要自由软件成果：
    - Emacs（文本编辑器）
    - GCC（GNU Compiler Collection，编译器）
    - GNU C Library 等工具链
  - Stallman 技术能力不如 Ken Thompson、Dennis Ritchie、Tanenbaum，但通过策略性发展推动了自由软件运动
- **自由软件运动与 GPL**
  - Stallman 创立 FSF（Free Software Foundation，自由软件基金会）
  - GPL（GNU General Public License）：保证软件自由使用、修改和分发的许可证
  - "自由"不等于"免费"：强调用户自由而非价格
- **Linux 的诞生**
  - **Linus Torvalds**：芬兰赫尔辛基大学学生
  - 基于 MINIX 的教学启发，独立编写 Linux 内核
  - 采用 GPL 许可证发布，结合 GNU 工具链形成完整操作系统
  - GNU/Linux：GNU 提供用户空间工具，Linux 提供内核

## 重要对比/关系

| 人物 | 贡献 | 特点 |
|------|------|------|
| Ken Thompson | 用汇编写出初版 UNIX | 技术能力极强，两个月完成 |
| Dennis Ritchie | 发明 C 语言，用 C 重写 UNIX | 解决可移植性问题 |
| Andrew Tanenbaum | 独立编写 MINIX 教学系统 | 仅用于教学，拒绝商用 |
| Richard Stallman | 发起 GNU 项目和自由软件运动 | 策略性发展，Emacs 破局 |
| Linus Torvalds | 编写 Linux 内核 | 受 MINIX 启发，GPL 发布 |

## 要点总结

1. GNU 项目的核心理念是**自由软件**（Free Software），强调用户对软件的自由而非免费
2. Stallman 的策略：操作系统内核失败 → 编译器失败 → **Emacs 成功** → 以收入反哺项目
3. Linux = Linus 编写的内核 + GNU 项目的用户空间工具，两者结合形成完整操作系统
4. 从 UNIX → MINIX → Linux 的传承链清晰展示了操作系统的演化脉络
5. 开源/自由软件运动深刻改变了软件行业的商业模式和开发方式
