# 第三章 操作系统相关概念（4）——计算机系统结构

## 本讲主题

计算机硬件的总线结构、中断机制、I/O 结构（同步与异步）、DMA 以及存储层次体系。

## 核心知识点

- **计算机硬件的总线结构**
  - CPU 和内存直接连接到总线
  - I/O 设备通过**设备控制器（Device Controller）**连接到总线
  - 设备控制器：**按类别管理**（如 USB 控制器管理所有 USB 设备），含本地缓存和寄存器
  - CPU 与 I/O 设备**并发运行**（可能服务于不同进程）
- **中断（Interrupt）**——操作系统最重要的概念之一
  - **定义**：让 CPU 放下正在运行的进程，转而处理更紧急事件的机制
  - 操作系统是**中断驱动**的（Interrupt-Driven）
  - **中断处理流程**：
    1. 设备控制器完成操作后产生中断信号
    2. CPU 保存当前进程的**上下文（Context）**：寄存器（Register）+ 程序计数器（Program Counter）
    3. CPU 转去处理中断
    4. 处理完毕后恢复被中断进程的上下文，继续执行
  - **中断的四种类型**：
    - **程序中断**：程序执行过程中产生的事件
    - **时钟中断**：定时器到期（如进程时间片用完）
    - **硬件故障中断**：设备出错
    - **I/O 中断**：I/O 操作完成（每次键盘敲击、鼠标移动、磁盘读取都产生中断）
  - 类比：看书时朋友叫你打球 → 做书签保存位置 → 打球 → 回来从书签处继续
- **I/O 结构**
  - **同步 I/O（Synchronous I/O）**：
    - 进程发起 I/O 请求后进入**等待状态（Waiting）**
    - I/O 完成、设备控制器产生中断后进程才恢复执行
    - **一次只能申请一个 I/O 设备**
    - CPU 不等待（切换执行其他进程）
  - **异步 I/O（Asynchronous I/O）**：
    - 进程发起 I/O 请求后**继续执行**，不进入等待状态
    - I/O 完成时由设备控制器产生中断通知 CPU
    - **可同时申请多个 I/O 设备**
    - 需要**设备状态表（Device Status Table）**追踪每个 I/O 设备状态和等待进程
- **DMA（Direct Memory Access，直接内存访问）**
  - 问题：大量数据传输时，每个字符产生一次中断会严重降低 CPU 效率
  - 解决：DMA 硬件接管数据传输，**整块数据传输完毕后才产生一次中断**
  - 流程：CPU 将指令写入 DMA → DMA 配合设备控制器完成传输 → 数据直接写入内存 → DMA 产生中断通知 CPU
- **存储层次体系**（从快到慢、从贵到便宜）
  - **寄存器（Register）**：最快，封装在 CPU 内
  - **缓存（Cache）**：封装在 CPU 内（如 i3=6MB, i5=9MB, i7=12MB）
  - **内存/主存（Main Memory）**：CPU 唯一能直接访问的大容量存储器
    - 缺点：容量小、断电数据丢失（易失性）
  - **电子磁盘/固态硬盘（SSD）**
  - **磁盘（Magnetic Disk）**：二级存储，容量大、断电不丢失
  - **光盘（Optical Disk）**
  - **磁带（Magnetic Tape）**：最慢最便宜
  - 内存可视为磁盘的缓存，Cache 可视为内存的缓存
- **缓存机制（Caching）**
  - **Cache**（名词）：CPU 中的高速缓存硬件
  - **Caching**（机制）：数据从慢速存储迁移到快速存储的过程
  - 数据迁移路径：磁盘 → 内存 → Cache → 寄存器
  - 缓存设计要点：缓存大小、替换策略（何时换入/换出）

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 设备控制器 | Device Controller | 管理同类 I/O 设备的硬件，含本地缓存和寄存器 |
| 中断 | Interrupt | CPU 放下当前任务转处理紧急事件的机制 |
| 上下文 | Context | 进程的运行状态信息（寄存器、程序计数器等） |
| 程序计数器 | Program Counter (PC) | 指向下一条待执行指令在内存中地址的寄存器 |
| 同步 I/O | Synchronous I/O | 进程发起 I/O 后等待完成才继续执行 |
| 异步 I/O | Asynchronous I/O | 进程发起 I/O 后继续执行，完成时通过中断通知 |
| DMA | Direct Memory Access | 绕过 CPU 直接在 I/O 设备与内存间传输数据块的硬件 |
| 设备状态表 | Device Status Table | 记录每个 I/O 设备当前状态和等待进程的数据结构 |
| 缓存机制 | Caching | 数据从慢速存储向快速存储迁移的过程 |

## 重要对比/关系

| 对比项 | 同步 I/O | 异步 I/O |
|--------|---------|---------|
| 进程状态 | 发起 I/O 后进入等待 | 发起 I/O 后继续执行 |
| 并发 I/O 数 | 一次一个 | 可同时多个 |
| 需要设备状态表 | 不需要 | 需要 |
| CPU 利用率 | 进程等待，CPU 可运行其他进程 | 当前进程与 I/O 并发执行 |

| 存储层级 | 速度 | 容量 | 价格 | 易失性 |
|----------|------|------|------|--------|
| 寄存器 | 最快 | 极小 | 最贵 | 是 |
| Cache | 很快 | 小(MB级) | 很贵 | 是 |
| 内存 | 快 | 中(GB级) | 贵 | 是 |
| SSD/磁盘 | 慢 | 大(TB级) | 便宜 | 否 |
| 磁带 | 最慢 | 最大 | 最便宜 | 否 |

## 要点总结

1. **中断是操作系统的核心驱动机制**：每秒产生大量中断，通过保存/恢复上下文实现进程切换
2. 同步 I/O 与异步 I/O 的根本区别在于**进程发起 I/O 后是否进入等待状态**
3. **DMA** 解决了大块数据传输时频繁中断 CPU 的问题，将"每字符一次中断"降为"每块一次中断"
4. 存储层次体系的核心原则：**越靠近 CPU 速度越快、容量越小、价格越贵**
5. **Caching 机制**贯穿整个存储体系：数据从磁盘→内存→Cache→寄存器逐级迁移
