# 第三章 操作系统相关概念（3）——操作系统分类（下）

## 本讲主题

个人计算机系统、多处理器系统（并行系统、分布式系统、集群系统）以及专用系统（实时系统、手持系统）的分类与特点。

## 核心知识点

- **个人计算机系统（PC OS）**
  - 面向单用户
  - 设计原则：**方便性（Convenience）** + **响应速度（Response）**
  - 代表：Windows、macOS、Linux
- **多处理器系统**
  - **并行系统（紧耦合系统，Tightly Coupled）**
    - 多个处理器在**同一主板**上，共享内存和时钟
    - 优势：增加吞吐量、提高可靠性（CPU 故障可替补）、经济优势（多 CPU 共建一个系统比各建一个系统便宜）
    - **对称结构（SMP, Symmetric Multiprocessing）**：每个处理器地位相同，都可运行操作系统和应用程序；现代 PC/手机基本都是此结构（i3/i5/i7 多核心）
    - **非对称结构（Asymmetric）**：一个主处理器（Master）+ 多个从处理器（Slave）；主处理器负责接收和分配作业，从处理器执行应用程序；一般用于大型计算机系统
  - **分布式系统（松耦合系统，Loosely Coupled）**
    - 多个主机分布在**广域网**上，**不共享内存**
    - 用途：资源共享、科学计算加速、提高可靠性（故障对用户不可见）、通信
    - 网络结构：
      - **Client-Server（C/S）**：客户端请求，服务器响应
      - **Peer-to-Peer（P2P）**：每个节点既是客户端也是服务器（如六维空间/BT 下载）
  - **集群系统（Cluster）**
    - 介于紧耦合与松耦合之间
    - **不共享内存，但共享存储**
    - 一般用于实验室/科学计算
    - **非对称集群（热备份模式）**：两台服务器成对运行，一台工作一台监视；工作机故障时备份机立即接管
    - **对称集群**：所有处理器共同运行应用
- **实时系统（Real-Time System）**
  - 应用：工业控制、科学实验、医疗设备、安全监控
  - 核心要求：在**限定时间内**做出响应
  - **硬实时系统（Hard Real-Time）**：
    - 必须在预定义时间内完成响应，否则视为系统失败
    - 一般为专有系统，不使用通用操作系统（如 Windows）
    - 应用：工业喷漆、化学滴定控制
  - **软实时系统（Soft Real-Time）**：
    - 不承诺在确定时间内完成，只保证给予**较高优先级**，尽快完成
    - 通用操作系统可支持（如视频聊天在众多进程中获高优先级）
- **手持系统（Handheld System）**
  - 早期 PDA（Personal Digital Assistant），现代指智能手机
  - 代表：Android、iOS
  - 特点：快速响应（触控）、高续航要求、内置多种传感器（距离、陀螺仪、方向等）

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 紧耦合系统 | Tightly Coupled System | 多处理器共享内存和时钟的并行系统 |
| 松耦合系统 | Loosely Coupled System | 多主机分布在广域网上、不共享内存的分布式系统 |
| SMP | Symmetric Multiprocessing | 对称多处理，所有处理器地位相同 |
| P2P | Peer-to-Peer | 对等网络，每个节点既是客户端也是服务器 |
| 热备份 | Hot Standby | 备用服务器处于运行状态监视主服务器，故障时立即接管 |
| 硬实时 | Hard Real-Time | 必须在严格时限内响应的实时系统 |
| 软实时 | Soft Real-Time | 尽力在短时间内响应但不做绝对保证的实时系统 |

## 重要对比/关系

| 系统类型 | 内存共享 | 分布范围 | 典型应用 |
|----------|---------|---------|---------|
| 并行系统（紧耦合） | 共享 | 同一主板/机箱 | 现代多核 PC/手机 |
| 分布式系统（松耦合） | 不共享 | 广域网 | 门户网站、CDN |
| 集群系统 | 不共享内存，共享存储 | 实验室/局域网 | 科学计算、高可用服务 |

| 对比项 | 硬实时 | 软实时 |
|--------|--------|--------|
| 时间保证 | 必须在预定义时限内 | 尽力而为，不做绝对承诺 |
| 系统类型 | 专有系统 | 通用操作系统可支持 |
| 失败后果 | 系统失败/事故 | 体验下降（如视频卡顿） |
| 应用场景 | 工业控制、医疗 | 视频通话、多媒体播放 |

## 要点总结

1. 并行系统的核心特征是**共享内存和时钟**，现代多核处理器均属对称多处理结构
2. 分布式系统的两种网络架构：**C/S**（集中式服务器）和 **P2P**（去中心化）
3. 集群系统的非对称结构即**热备份模式**，是提高服务可靠性的重要手段
4. **硬实时**要求绝对时限保证（超时即失败），**软实时**只保证高优先级尽快执行
5. 手持系统设计需兼顾响应速度、电池续航和传感器管理
