# 第三章 操作系统相关概念（2）——操作系统分类（上）

## 本讲主题

操作系统的分类体系概览，以及大型机操作系统的三种形态：批处理系统、多道程序系统和分时系统。

## 核心知识点

- **操作系统七大分类**
  1. 大型机操作系统（Mainframe）：批处理、多道程序、分时
  2. 桌面操作系统（Desktop/PC）
  3. 多处理器系统：并行系统（紧耦合）、分布式系统（松耦合）、集群系统
  4. 实时系统（Real-Time）
  5. 手持系统（Handheld）
- **批处理系统（Batch System）**
  - 出现时间：20 世纪 50 年代（第二代计算机，晶体管）
  - 特点：
    - 一个时间段内**只能执行一个作业**
    - 自动从一个作业切换到下一个作业
  - 工作流程：程序员准备代码 + 数据 + 控制信息 → 提交管理员 → 批量运行 → 返回结果
  - **核心问题**：CPU 经常空闲（进程在 I/O 区间时 CPU 无事可做）
- **CPU 区间与 I/O 区间（CPU Burst & I/O Burst）**（重要概念）
  - 程序运行交替经历 CPU 区间和 I/O 区间
  - **I/O 密集型程序（I/O-Bound）**：I/O 区间 > CPU 区间（大多数程序）
  - **CPU 密集型程序（CPU-Bound）**：CPU 区间 > I/O 区间（科学计算类，如龙格-库塔法计算微分方程）
  - CPU 速度远高于 I/O 设备速度 → 批处理系统 CPU 利用率低
- **多道程序系统（Multiprogramming System）**
  - 核心思想：**同时将多个作业加载到内存**，当一个进程进入 I/O 区间时，CPU 切换执行另一个进程
  - 关键技术：磁盘技术引入 + **作业池（Job Pool）**概念
  - 内存状态示例：操作系统占低地址区间，剩余空间分配给 Job 1~4
  - **两种调度**：
    - **作业调度（Job Scheduling / 长期调度）**：从作业池选择子集加载到内存
    - **CPU 调度（CPU Scheduling / 短期调度）**：从内存中选择进程分配 CPU
  - **调度（Scheduling）**：当多个竞争者争抢某类资源时，按规则规定使用顺序的过程
    - 经典算法：先到先服务（FCFS, First Come First Served）
  - **内存保护**：通过授权机制保证进程运行互不影响，每个进程只能访问操作系统分配给自己的内存区域
- **分时系统（Time-Sharing System）**
  - 多道程序系统的逻辑扩展
  - 特点：**多用户** + **交互性**
  - 原理：将内存中不同进程分配给不同用户，CPU 快速轮转，每个用户感觉独占计算机
  - 利用 CPU 区间与 I/O 区间的速度差异，为用户提供充足的交互时间

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| CPU 区间 | CPU Burst | 进程在 CPU 上执行计算的时间段 |
| I/O 区间 | I/O Burst | 进程等待 I/O 操作完成的时间段 |
| I/O 密集型 | I/O-Bound | I/O 区间远大于 CPU 区间的程序类型 |
| CPU 密集型 | CPU-Bound | CPU 区间远大于 I/O 区间的程序类型 |
| 作业池 | Job Pool | 磁盘上保存所有待运行作业的区域 |
| 作业调度 | Job Scheduling | 从作业池选择作业子集加载到内存的过程（长期调度） |
| CPU 调度 | CPU Scheduling | 从内存中选择进程分配 CPU 的过程（短期调度） |
| 多道程序 | Multiprogramming | 内存中同时存在多个程序，交替使用 CPU |
| 分时系统 | Time-Sharing System | 多用户共享计算机，CPU 快速轮转实现交互 |

## 重要对比/关系

| 系统类型 | 并发作业数 | 用户交互 | CPU 利用率 | 典型时代 |
|----------|-----------|---------|-----------|---------|
| 批处理系统 | 1 | 无 | 低 | 第二代计算机 |
| 多道程序系统 | 多个 | 无 | 高 | 第三代计算机 |
| 分时系统 | 多个 | 有（多用户） | 高 | 第三代计算机 |

## 要点总结

1. **CPU Burst 与 I/O Burst** 是贯穿操作系统课程的核心概念，决定了进程调度策略
2. 批处理系统的核心缺陷是 **CPU 在 I/O 区间空闲**，多道程序系统通过多作业并发解决此问题
3. 多道程序系统引入两级调度：**作业调度**（磁盘→内存）和 **CPU 调度**（内存→CPU）
4. 分时系统是多道程序系统的逻辑扩展，核心特征是**多用户交互**
5. 内存保护通过授权机制实现，确保进程只能访问自己被分配的内存区域
