# 第三章 操作系统相关概念（6）——操作系统服务与系统调用

## 本讲主题

操作系统向上层提供的服务种类，以及系统调用（System Call）的概念、参数传递方式和系统调用接口的工作机制。

## 核心知识点

- **操作系统提供的服务**
  1. **程序执行**：将用户程序加载到内存，参与调度和分配
  2. **I/O 操作**：向上层程序提供使用 I/O 资源的方式
  3. **文件管理**：文件与目录的读写、创建、删除
  4. **进程通信（IPC）**：
     - **共享内存（Shared Memory）**：同一计算机内多进程间通信
     - **消息传递（Message Passing）**：适用于广域网/分布式系统，也可用于本地进程
  5. **错误检测**：检测用户进程错误并采取相应措施
  6. **辅助功能**：
     - 资源分配
     - 记账/统计信息（记录进程资源使用量和时间）
     - 安全保护（确保资源以授权方式使用）
- **系统调用（System Call）**
  - **定义**：正在运行的用户程序与操作系统之间的接口
  - 通过系统调用使用操作系统提供的服务
  - **API 与系统调用的关系**：
    - 程序员学习和使用的是 **API**（Application Program Interface）
    - API 由运行时函数库（Runtime Library）转换为底层的系统调用
    - 不直接使用系统调用的原因：系统调用参数多（可达几十个）、直接使用有安全隐患
  - **参数传递的三种方式**：
    1. **寄存器传递**：参数少时，写入 CPU 寄存器（用户模式写入 → 内核模式读取）
    2. **内存表传递**：参数多时，在内存中建表存放参数，将表的基地址写入寄存器
    3. **栈传递**：通过压栈（Push）和出栈（Pop）传递参数
  - **系统调用接口（System Call Interface）**
    - 维护一个系统调用表：第一列为系统调用编号，第二列为处理程序的位置
    - 流程：接收请求 → 判断编号 → 激活对应处理程序 → 返回结果给用户进程
  - **系统调用覆盖的功能**：进程管理、文件管理、设备管理、信息管理、通信
- **系统程序（System Programs）**
  - 位于操作系统之上、用户应用程序之下
  - 有时是系统调用的简单包装，有时非常复杂
  - 功能：文件管理、状态信息管理、文件修改、编程语言支持等
  - 系统程序更贴近操作系统，用户应用程序更贴近用户

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 系统调用 | System Call | 用户程序请求操作系统服务的接口机制 |
| API | Application Program Interface | 程序员使用的应用程序编程接口，由运行时库转为系统调用 |
| 共享内存 | Shared Memory | 多进程通过共享内存区域进行通信的方式 |
| 消息传递 | Message Passing | 进程间通过发送和接收消息进行通信的方式 |
| 系统调用接口 | System Call Interface | 维护系统调用编号与处理程序映射表的中间层 |
| 系统程序 | System Program | 介于操作系统和用户程序之间的程序，提供操作系统服务的高级封装 |

## 重要对比/关系

| 对比项 | API | 系统调用 |
|--------|-----|---------|
| 使用者 | 程序员 | 操作系统内部 |
| 参数数量 | 少（几个） | 多（可达几十个） |
| 安全性 | 受控 | 直接操作硬件，风险高 |
| 转换关系 | API → 运行时库 → 系统调用 | — |

| 通信方式 | 共享内存 | 消息传递 |
|----------|---------|---------|
| 适用范围 | 同一计算机内 | 广域网/本地均可 |
| 通信机制 | 进程共享内存区域 | 发送/接收消息 |
| 系统类型 | 紧耦合系统 | 松耦合/紧耦合均可 |

## 要点总结

1. **系统调用**是用户程序使用操作系统服务的唯一正式接口
2. 程序员通过 **API** 编程，运行时库自动将 API 转换为底层系统调用
3. 参数传递有三种方式：**寄存器**（少量）、**内存表**（大量）、**栈**
4. 系统调用接口通过**编号映射表**管理所有系统调用的分发
5. 进程间通信的两大方式：**共享内存**（同机）和**消息传递**（跨机/同机）
