# 第一章 绪论

## 本讲主题

操作系统课程导论：介绍操作系统的基本概念、课程结构和先修知识要求。

## 核心知识点

- **操作系统的地位**
  - 软件工程/计算机科学的基础理论课、必修课、学位课
  - 实际工作中涉及死锁、并发计算等核心概念
  - 考研必考科目（与数据结构、组成原理、软件工程并列）
- **什么是操作系统（感性认识）**
  - 向下：管理底层硬件（CPU、内存、主板等）
  - 向上：为用户提供友好的用户界面
  - 现代软件工程的里程碑
- **课程三大模块**
  - **进程管理**
    - 进程/线程基本概念
    - 进程通信与同步
    - CPU 调度（Windows 10 开机约 150 个进程、2600 个线程）
  - **内存管理**
    - 冯·诺依曼体系：程序必须加载到内存才能运算
    - 为多个进程分配有限内存
  - **I/O 管理**
    - 管理硬盘、键盘、鼠标、显示器、打印机等硬件设备
    - 设备与进程之间的通信
- **冯·诺依曼（Von Neumann）贡献**
  - 将运算逻辑（软件）与硬件分离——革命性思想
  - 提出二进制计算体系
  - 程序作为数据存储在外存，运行时调入内存
- **先修知识**
  - 计算机组成原理
  - 至少一种高级编程语言（C/Java 等）
  - 软件设计模式（了解即可）
  - 基础数据结构（线性表、链表等，需熟练掌握）

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 操作系统 | Operating System (OS) | 管理计算机硬件资源、为用户和应用程序提供服务的系统软件 |
| 冯·诺依曼体系 | Von Neumann Architecture | 将程序和数据存储在存储器中，按地址顺序执行指令的计算机体系结构 |
| 进程 | Process | 正在执行的程序实例 |
| 线程 | Thread | 进程内的执行单元，共享进程资源 |
| CPU 调度 | CPU Scheduling | 在多个进程间分配 CPU 时间的策略 |

## 要点总结

1. 操作系统的核心功能是**向下管理硬件、向上服务用户**，起到承上启下的衔接作用
2. 课程三大模块：**进程管理**（含 CPU 调度）、**内存管理**、**I/O 管理**
3. 冯·诺依曼体系的核心贡献是**软硬件分离**和**存储程序**概念
4. 学习本课程需具备组成原理、编程语言和数据结构基础
5. 推荐教材：*Operating System Concepts*（恐龙书/龙宝书，第九版）
