# 第三章 操作系统相关概念（5）——操作系统的组成

## 本讲主题

操作系统的八大组件：进程管理、内存管理、文件管理、I/O 系统、二级存储管理、网络、安全保护和命令行解释程序。

## 核心知识点

- **操作系统八大组件**
  1. **进程管理（Process Management）**
     - 进程 = 正在执行的程序（1966 年 Multics 项目中由 MIT 工程师首次提出）
     - 进程只在**多道程序环境（并发环境）**下才有意义
     - 进程执行需要的资源：CPU、内存、I/O、文件、数据
     - 进程管理功能：创建、删除、停止、恢复、同步、通信
  2. **内存管理（Memory Management）**
     - 内存 = 巨大的字/字节数组，每个单元有绝对地址
     - CPU 唯一能直接访问的大容量存储器
     - 为 CPU 和 I/O 设备提供数据交换场所
     - 功能：记录内存使用状态、决定进程加载、分配与释放内存
  3. **文件管理（File Management）**
     - **文件**：逻辑上的信息存储单元，向下屏蔽存储介质异构性（磁信号/光信号/电信号）
     - 文件 = 相关信息的集合，由创建者定义
     - 功能：文件/目录的创建与删除、提供操作文件的**原语（Primitive）**、文件到二级存储的映射、备份与恢复
     - **原语（Primitive）**：实现某功能的指令集合，执行过程中**不允许中断**（要么全部执行，要么全不执行）
  4. **I/O 系统管理**
     - **缓存（Caching）**：数据从慢速存储迁移到快速存储的机制
     - **缓冲（Buffer/Buffering）**：在内存中开辟区域，解决设备间速度差异
       - 例：Modem（56Kbps）→ 缓冲区 → 磁盘（100MB/s），先积累数据再批量写入
     - 提供通用的驱动程序接口（新硬件按接口装驱动即可使用）
     - 管理所有 I/O 设备的驱动程序
  5. **二级存储管理（Secondary Storage Management）**
     - 内存两大缺陷：断电丢失 + 容量小
     - 磁盘作为二级存储：容量大、断电不丢失
     - 功能：空闲空间管理、存储空间分配、磁盘调度
  6. **网络管理**
     - 分布式系统（松耦合）：通过网络连接，共同完成工作
     - 通过协议通信（如 TCP/IP）
     - 作用：资源共享、计算加速
  7. **安全保护系统（Protection System）**
     - 用户进程不能干扰操作系统进程
     - 进程不能访问其他进程的内存地址
     - **硬件模式位（Mode Bit）**机制：
       - **0 = 内核模式（Kernel Mode）**：可执行特权指令，运行操作系统进程
       - **1 = 用户模式（User Mode）**：运行用户进程
     - **特权指令（Privileged Instruction）**：对硬件操作等可能产生危险的指令，必须在内核模式下执行
     - 开机流程：内核模式（引导 + 初始化）→ 用户模式（运行用户程序）→ 需要资源时通过系统调用切回内核模式
     - 资源使用必须先向操作系统**申请授权**
  8. **命令行解释程序（Command-Line Interpreter / Shell）**
     - Linux/UNIX：Bourne Shell、C-Shell、Korn Shell 等
     - 两种实现方式：内核中实现 / 系统程序实现（更灵活、易扩展）
     - 功能：进程创建/管理、I/O 管理、磁盘管理等

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 进程 | Process | 多道程序环境下正在执行的程序 |
| 原语 | Primitive | 不可中断的指令集合，要么全部执行要么全不执行 |
| 缓冲 | Buffer/Buffering | 在内存中开辟区域解决设备间速度差异的机制 |
| 缓存 | Caching | 数据从慢速存储向快速存储迁移的机制 |
| 模式位 | Mode Bit | CPU 中区分内核模式(0)和用户模式(1)的硬件位 |
| 内核模式 | Kernel Mode | CPU 可执行特权指令的运行模式 |
| 用户模式 | User Mode | CPU 运行用户程序的受限模式 |
| 特权指令 | Privileged Instruction | 只能在内核模式下执行的危险/敏感指令 |
| Shell | Shell | 命令行解释程序，接收用户命令并调用系统服务 |

## 重要对比/关系

| 概念 | 缓冲（Buffer） | 缓存（Caching） |
|------|----------------|-----------------|
| 目的 | 解决设备间**速度差异** | 加速数据访问 |
| 位置 | 内存中开辟的特定区域 | 快速存储设备（Cache、内存等） |
| 机制 | 数据积累后批量传输 | 数据从慢存储向快存储迁移 |
| 示例 | Modem 数据先写缓冲再写磁盘 | 数据从内存拷贝到 CPU Cache |

| 模式 | 内核模式 | 用户模式 |
|------|---------|---------|
| Mode Bit | 0 | 1 |
| 可执行指令 | 全部（含特权指令） | 仅普通指令 |
| 运行内容 | 操作系统进程 | 用户进程 |
| 安全级别 | 最高 | 受限 |

## 要点总结

1. 操作系统由**八大组件**构成，每个组件对应一类核心功能
2. **原语**是不可中断的执行单元，是操作系统实现原子操作的基础
3. **缓冲（Buffer）**解决设备速度差异，**缓存（Caching）**加速数据访问——两个概念不同
4. 安全保护通过**模式位**实现双模式切换：内核模式执行特权指令，用户模式执行普通指令
5. 所有资源使用必须通过操作系统**授权**，未授权的资源用户进程不可使用
