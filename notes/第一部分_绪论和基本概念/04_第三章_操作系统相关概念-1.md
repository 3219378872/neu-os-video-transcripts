# 第三章 操作系统相关概念（1）——操作系统的定义

## 本讲主题

操作系统的正式定义：从计算机系统的四层结构出发，明确操作系统作为资源管理者、资源分配者和程序运行管理者的三重角色。

## 核心知识点

- **计算机系统的四层结构**（从下到上）
  1. **硬件层**：CPU、内存、主板、I/O 设备（键盘、鼠标、打印机、显示器等）
  2. **操作系统层**：管理硬件，为上层提供服务，承上启下
  3. **应用程序层**：Office 系列、编译器、游戏等用户安装的软件
  4. **用户层**：最终使用计算机的人
- **操作系统的定义**
  - 操作系统是计算机的**管理程序**
  - 向下管理计算机的**硬件资源**
  - 向上为应用程序提供**运行环境**，为用户提供**简洁方便的用户界面**
  - 是底层系统**资源的分配器**
  - 是**程序运行的管理程序**
- **操作系统的三个目标**
  - **执行用户程序**：协调多个程序互不干扰地运行
  - **方便使用**：提供友好的用户界面（Windows 与 macOS 走不同的设计路径）
  - **资源分配**：为众多并发进程公平公正地分配 CPU、内存等有限资源
- **操作系统作为管理程序**
  - 管理程序运行，避免错误产生
  - 错误应限定在产生错误的程序内部，不影响操作系统运行

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 操作系统 | Operating System | 管理计算机硬件资源，向上提供运行环境和用户界面的系统软件 |
| 资源分配器 | Resource Allocator | 在多个竞争进程间分配有限计算资源的角色 |
| 管理程序 | Control Program | 管理程序运行、防止错误和非法使用的系统组件 |
| 用户界面 | User Interface (UI) | 操作系统提供给用户的交互方式 |

## 要点总结

1. 计算机系统分四层：**硬件 → 操作系统 → 应用程序 → 用户**
2. 操作系统的核心定义：**管理硬件资源 + 提供运行环境 + 分配资源 + 管理程序运行**
3. 操作系统的三重角色：硬件的**管理者**、资源的**分配者**、程序的**管理程序**
4. 操作系统必须确保一个程序的错误不会影响到其他程序或操作系统本身
