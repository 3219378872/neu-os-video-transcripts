# 第二章 操作系统的历史（上）

## 本讲主题

操作系统的发展历程：从无操作系统时代到 UNIX 诞生，以及 AT&T 版权收回对教育界的深远影响。

## 核心知识点

- **计算机代际与操作系统演进**
  - **第一代（1946–1958）**：电子管计算机，无操作系统，打孔输入
  - **第二代（1958–1964）**：晶体管计算机，出现批处理（Batch）系统雏形，一次只能运行一个作业
  - **第三代（1964–）**：集成电路，现代操作系统雏形，典型代表 IBM 360 系统
  - **第四代**：大规模/超大规模集成电路，现代操作系统特征（多任务、多媒体、更好的 UI）
- **计算机先驱**
  - **冯·诺依曼（Von Neumann）**：计算机之父、博弈论之父，提出软硬件分离和二进制
  - **阿兰·图灵（Alan Turing）**：数学家、密码学家，二战密码破译
- **UNIX 的诞生**
  - **CTSS 分时系统**：MIT 开发，支持约 30 个终端共享大型机（Compatible Time-Sharing System）
  - **Multics 项目（1965）**：MIT + 贝尔实验室 + 通用电气，目标支持 300+ 终端，最终失败（超预算、超时——软件危机典型案例）
  - **Ken Thompson（1969）**：贝尔实验室工程师，用汇编语言在 DEC PDP-7 上用两个月写出初版 UNIX（为移植太空旅游游戏）
  - **Dennis Ritchie**：先用 B 语言重写 UNIX（效率差），后发明 **C 语言**重写 UNIX，解决可移植性问题
- **UNIX 的扩散**
  - AT&T 早期态度开放（不支持也不阻止），促进 UNIX 蓬勃发展
  - 商业发行版：AT&T System V、IBM AIX、Sun Solaris、HP-UX
  - **BSD（Berkeley Software Distribution）**：Bill Joy 在伯克利大学基于 UNIX 开发，加入编译器和科学计算软件；后创办 Sun 公司
- **AT&T 版权收回（1979）**
  - System V 第七版发布后宣布 UNIX 为版权软件
  - 禁止大学教师在教学中免费使用 UNIX 代码
  - 直接催生 MINIX 和后续 Linux 的诞生
- **MINIX 的诞生**
  - **Andrew S. Tanenbaum**（谭宁邦）：荷兰阿姆斯特丹大学教授
  - 在完全未参考 UNIX 源码的情况下独立编写 MINIX，仅用于教学
  - MINIX 直接催生了 Linux 的诞生

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 分时系统 | Time-Sharing System | 多用户共享一台计算机，CPU 快速轮转使每个用户感觉独占系统 |
| 批处理系统 | Batch System | 一次提交一批作业，顺序执行，无交互 |
| 软件危机 | Software Crisis | 1968 年提出，指软件项目普遍超预算、超时、bug 多、维护困难的现象 |
| 可移植性 | Portability | 软件在不同硬件/平台上运行的能力 |
| BSD | Berkeley Software Distribution | 加州大学伯克利分校基于 UNIX 开发的操作系统发行版 |
| MINIX | Mini-UNIX | Tanenbaum 教授为教学目的独立开发的类 UNIX 操作系统 |

## 重要对比/关系

| 项目 | CTSS | Multics | UNIX |
|------|------|---------|------|
| 时间 | ~1960 | 1965–1969 | 1969 |
| 开发方 | MIT | MIT + 贝尔 + GE | 贝尔实验室（Ken Thompson） |
| 终端支持 | ~30 | 目标 300+ | 初期单用户 |
| 结果 | 成功 | 失败 | 成功，影响深远 |

## 要点总结

1. 操作系统的发展与硬件代际紧密相关：电子管→晶体管→集成电路→大规模集成电路
2. UNIX 诞生于 Multics 失败之后，Ken Thompson 用两个月汇编代码改变了计算机历史
3. C 语言的发明直接源于重写 UNIX 的需求（Dennis Ritchie）
4. AT&T 1979 年收回 UNIX 版权，禁止教育界免费使用，催生了 MINIX 和 Linux
5. 计算机史上的牛人面对"没有操作系统"的回答惊人一致：**自己写一个**
