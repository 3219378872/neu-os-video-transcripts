# 第十章 文件系统——挂载和保护

## 本讲主题

介绍文件系统的挂载机制与基于拥有者和组的文件保护方案。

## 核心知识点

- **文件系统挂载**
  - 文件系统在使用前必须挂载（Mount）到当前系统中
  - 挂载点（Mount Point）是新文件系统接入的位置
  - 新文件系统挂载后替换原挂载点下的目录内容，原目录可隐藏或删除
  - 广泛应用于操作系统和分布式系统
- **文件保护机制**
  - 基于拥有者（Owner）和组（Group）的权限模型
  - 拥有者：创建文件的用户，负责定义文件的访问权限
  - 组：具有相同权限的用户集合，一个文件可关联多个组（读组、写组等）
- **权限类型**
  - 读（Read）、写（Write）、执行（Execute）
  - 追加（Append）、删除（Delete）、列举（List）
  - 不同身份的用户对应不同的操作权限
- **访问控制列表**
  - Windows 系统使用 ACL（Access Control List）管理权限
  - ACL 定义不同用户和组对文件的具体操作权限

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 挂载 | Mount | 将文件系统接入当前系统目录树的操作 |
| 挂载点 | Mount Point | 新文件系统接入的目录位置 |
| 拥有者 | Owner | 创建文件的用户，拥有权限定义权 |
| 组 | Group | 具有相同访问权限的用户集合 |
| 访问控制列表 | ACL (Access Control List) | 定义用户和组对文件操作权限的列表 |

## 要点总结

1. 文件系统使用前必须通过**挂载**接入当前系统，挂载点决定接入位置
2. 文件保护基于**拥有者和组**模型，由文件创建者定义访问权限
3. 权限操作包括**读、写、执行、追加、删除、列举**六种
4. 组是具有相同权限的**用户集合**，一个文件可关联不同权限的组
5. Windows 使用 **ACL** 机制精细控制不同用户和组的文件访问权限
