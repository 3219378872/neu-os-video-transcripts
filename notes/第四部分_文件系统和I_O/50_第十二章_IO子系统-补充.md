# 第十二章 I/O子系统——补充

## 本讲主题

介绍 I/O 请求从应用程序到硬件操作的转换流程。

## 核心知识点

- **I/O 请求转换流程**
  - 应用程序发起文件读取请求（如读取 C:\h\a.txt）
  - 第一步：通过设备表查找目标盘符对应的驱动器和端口号
  - 第二步：物理读取对应驱动器上的数据
  - 第三步：将数据加载到内存中
  - 第四步：将 CPU 控制权交还给发起请求的进程
- **设备表的作用**
  - 维护盘符到物理驱动器的映射关系
  - 记录端口号、驱动器编号等硬件信息
  - 是 I/O 请求转换为硬件操作的关键查找结构
- **I/O 操作的整体路径**
  - 应用程序 → 逻辑文件系统 → 文件组织模块 → 基础文件系统 → I/O 控制层 → 硬件
  - 与文件系统分层结构一致，逐层转换直至硬件执行

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 设备表 | Device Table | 记录盘符到物理驱动器映射关系的表 |
| I/O 请求 | I/O Request | 应用程序发起的数据读写操作请求 |
| 驱动器 | Drive | 执行物理读写操作的硬件设备 |
| 端口号 | Port Number | 标识设备连接位置的编号 |

## 要点总结

1. I/O 请求需通过**设备表**查找盘符对应的物理驱动器和端口号
2. 数据从磁盘**物理读取**后加载到内存，再将控制权交还进程
3. 整个过程遵循文件系统**分层结构**，从逻辑层逐步转换到硬件操作
4. **设备表**是连接逻辑文件路径与物理硬件位置的关键桥梁
5. I/O 请求转换流程体现了操作系统**层层抽象**的设计思想
