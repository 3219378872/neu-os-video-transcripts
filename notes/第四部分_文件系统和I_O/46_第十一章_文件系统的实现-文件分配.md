# 第十一章 文件系统的实现——文件分配

## 本讲主题

介绍文件到磁盘的三种分配方法：连续分配、链接分配和索引分配。

## 核心知识点

- **连续分配**
  - 在磁盘中找到一组连续块分配给文件，记录起始块号和文件长度
  - 同时支持顺序访问和随机访问
  - 问题：难以预估文件大小、产生外部碎片、文件无法动态增长
  - 改进方案：使用扩展（Extent），不够时再分配一块连续空间
- **链接分配**
  - 文件占用的磁盘块不连续，每个块包含数据和指向下一块的指针
  - 解决了连续分配的碎片问题和文件增长问题
  - 问题：指针断裂导致后续数据丢失；仅支持顺序访问，不支持随机访问
  - 目录记录文件的起始块和结束块
- **FAT 文件分配表**
  - 链接分配的变形，建立全局 FAT（File Allocation Table）
  - 磁盘有 N 个块则 FAT 有 N 项，每项记录当前块的下一块号
  - 目录仅记录文件起始块号，通过 FAT 链式查找后续块
  - 末尾标记为 End of File；广泛应用于 MS-DOS 和 Windows 系统
- **索引分配**
  - 将文件所有块号集中保存在一个独立的索引块中
  - 目录仅记录索引块号，索引块内按序列出所有数据块号
  - 同时支持顺序访问和随机访问
  - 需要额外的索引块开销

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 连续分配 | Contiguous Allocation | 为文件分配一组连续磁盘块的方法 |
| 链接分配 | Linked Allocation | 每个块含数据和下一块指针的分配方法 |
| 索引分配 | Indexed Allocation | 用独立索引块记录文件所有块号的分配方法 |
| 扩展 | Extent | 连续分配的改进，按需追加连续空间 |
| 文件分配表 | FAT (File Allocation Table) | 全局表记录所有块间链接关系 |
| 外部碎片 | External Fragmentation | 文件间无法利用的小块空闲空间 |

## 重要对比/关系

| 对比项 | 连续分配 | 链接分配 | 索引分配 |
|--------|---------|---------|---------|
| 随机访问 | 支持 | 不支持 | 支持 |
| 顺序访问 | 支持 | 支持 | 支持 |
| 碎片问题 | 有外部碎片 | 无 | 无 |
| 文件增长 | 困难 | 容易 | 容易 |
| 额外开销 | 无 | 每块含指针 | 需索引块 |
| 可靠性 | 高 | 指针断裂风险 | 高 |

## 要点总结

1. **连续分配**记录起始块和长度，支持随机访问但存在碎片和增长问题
2. **链接分配**通过块内指针链接，解决碎片问题但仅支持顺序访问
3. **FAT** 是链接分配的改进，用全局表记录块间关系，广泛用于 **Windows** 系统
4. **索引分配**用独立索引块集中记录所有块号，兼顾随机和顺序访问
5. 三种方法各有优劣，现代文件系统常**组合使用**多种分配策略
