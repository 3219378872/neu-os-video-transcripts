# 第十二章 I/O子系统——通讯与调度

## 本讲主题

介绍内核 I/O 子系统提供的服务：调度、缓冲、缓存、假脱机及设备预留。

## 核心知识点

- **I/O 调度**
  - 多个进程竞争有限 I/O 资源时，需调度算法安排使用顺序
  - 每个 I/O 设备维护各自的等待队列
  - 目标：公平、高效地为等待进程分配 I/O 资源
- **缓冲区（Buffering）**
  - 为弥补 I/O 设备间的速度差异而在内存中开辟的区域
  - 示例：Modem 接收数据先写入 Buffer，积累成块后一次性写入磁盘
  - 解决数据传输量不匹配的问题
- **缓存（Caching）**
  - 数据从慢速存储向快速存储迁移的机制
  - 迁移路径：磁盘 → 内存 → CPU 缓存（Cache） → 寄存器
  - Cache 特指 CPU 的 L1/L2/L3 缓存硬件；Caching 是通用迁移机制
- **假脱机（Spooling）**
  - 针对打印机等非共享设备的解决方案
  - 打印请求先输出到文件，操作系统维护打印文件队列
  - 队列中的文件逐一输出到打印机，避免多任务打印数据混合
- **设备预留（Device Reservation）**
  - 进程使用 I/O 设备前向操作系统申请，由系统分配或预留
  - 适用于非共享的独占型 I/O 设备
  - 需特别注意死锁问题
- **内核 I/O 子系统的其他数据结构**
  - 打开文件表（系统范围和单进程）
  - 网络连接服务、字符集状态等

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| I/O 调度 | I/O Scheduling | 安排多进程使用 I/O 设备顺序的机制 |
| 缓冲区 | Buffer | 弥补设备速度差异的内存区域 |
| 缓存 | Caching | 数据从慢速到快速存储迁移的机制 |
| 假脱机 | Spooling | 将打印请求排队后逐一输出的技术 |
| 设备预留 | Device Reservation | 操作系统为进程预留独占设备的机制 |

## 重要对比/关系

| 对比项 | 缓冲（Buffering） | 缓存（Caching） |
|--------|-------------------|-----------------|
| 目的 | 弥补速度差异、匹配数据量 | 加速数据访问 |
| 数据位置 | 数据暂存于 Buffer | 数据副本存于快速存储 |
| 典型场景 | Modem → 磁盘 | 磁盘 → 内存 → CPU Cache |

## 要点总结

1. **I/O 调度**为等待队列中的进程公平分配设备资源，每个设备有独立队列
2. **缓冲区**在内存中开辟，用于弥补 I/O 设备间的速度和数据量差异
3. **缓存**是数据从慢速到快速存储的迁移机制，与 CPU 的 **Cache** 硬件是不同概念
4. **假脱机**将打印请求排队为文件，逐一输出，解决非共享设备的并发访问问题
5. 独占型设备使用**设备预留**机制，需防范死锁问题
