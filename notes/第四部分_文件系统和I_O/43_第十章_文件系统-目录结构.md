# 第十章 文件系统——目录结构

## 本讲主题

介绍磁盘与分区的关系、目录操作及五种目录组织结构。

## 核心知识点

- **磁盘、分区与卷**
  - 磁盘（Disk）是物理存储设备，分区（Partition）是逻辑磁盘
  - 一个物理磁盘可分为多个分区（如 1TB 分为 100G 系统盘 + 900G 数据盘）
  - 多个物理磁盘也可合并为一个分区
  - 卷（Volume）是采用文件系统管理的分区
- **目录的基本操作**
  - 查找文件（Search）、创建文件（Create）、删除文件（Delete）
  - 列举目录（List Directory）、重命名（Rename）、遍历文件系统（Traverse）
  - 遍历与列举不同：列举仅展示条目，遍历需逐一访问文件内容
- **单层目录**
  - 所有文件放在同一目录下，实现简单
  - 致命缺陷：文件名不能重复，多用户环境极易发生文件覆盖
- **双层目录**
  - 第一级为主文件目录（MFD），第二级为用户文件目录（UFD）
  - 每个用户拥有独立目录，解决了重名问题
  - 缺陷：不同用户之间无法共享文件或目录
- **树状目录**
  - 双层目录的多层扩展，每个叶子节点有唯一路径名
  - 当前操作系统最常用的目录结构
  - 仍无法解决多用户共享文件的需求
- **无环图目录**
  - 基于树状结构，允许不同用户共享同一文件
  - 问题：同一文件可能有多个路径名；需处理共享文件的删除策略
- **通用图目录**
  - 允许目录结构中出现环，是不推荐的结构
  - 环会导致遍历文件系统时陷入死循环
  - 每新增链接需运行环检测算法

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 分区 | Partition | 逻辑磁盘，由物理磁盘划分而来 |
| 卷 | Volume | 采用文件系统管理的分区 |
| 主文件目录 | MFD (Master File Directory) | 双层目录中的第一级，管理所有用户目录 |
| 用户文件目录 | UFD (User File Directory) | 双层目录中的第二级，每用户一个 |
| 无环图 | Acyclic Graph | 允许共享但不含环的目录结构 |
| 路径名 | Path Name | 从根到文件的唯一标识路径 |

## 重要对比/关系

| 目录结构 | 重名解决 | 文件共享 | 主要问题 |
|---------|---------|---------|---------|
| 单层目录 | 不支持 | 不支持 | 文件名冲突 |
| 双层目录 | 支持 | 不支持 | 无法跨用户共享 |
| 树状目录 | 支持 | 不支持 | 无法跨用户共享 |
| 无环图目录 | 支持 | 支持 | 多路径名、删除策略 |
| 通用图目录 | 支持 | 支持 | 环导致死循环 |

## 要点总结

1. **磁盘**是物理设备，**分区**是逻辑磁盘，**卷**是被文件系统管理的分区
2. **单层目录**实现简单但无法解决多用户重名问题，双层目录为每用户建独立目录
3. **树状目录**是当前最常用结构，每个文件有唯一路径名
4. **无环图目录**允许共享文件，但需处理多路径名和删除策略问题
5. 目录结构中必须**避免环**的出现，新增链接时需进行环检测
