# 第十一章 文件系统的实现——文件系统结构

## 本讲主题

介绍文件系统的分层结构、磁盘上的关键数据结构及文件的创建/打开/关闭流程。

## 核心知识点

- **磁盘特性**
  - 磁盘可重写，支持反复擦写
  - 支持直接访问（随机访问），可定位任意块进行读取
  - 以块（Block）为基本组织单位，典型值为 512B 或 4KB
- **文件系统分层结构（四层）**
  - I/O 控制层：包含设备驱动程序和中断机制，完成内存与磁盘间的数据传输
  - 基础文件系统（Basic File System）：生成命令发送给驱动程序，指定驱动器、柱面、磁道、扇区
  - 文件组织模块（File Organization Module）：将文件逻辑块映射到物理块
  - 逻辑文件系统（Logical File System）：管理元数据（Meta Data），即目录结构和文件信息
- **磁盘上的关键数据结构**
  - 引导控制块（Boot Control Block）：记录操作系统在磁盘上的位置，供引导程序定位
  - 卷控制块（Volume Control Block）：记录卷的块总数、块大小、空闲块和已占用块等信息
  - 目录结构：Linux/Unix 使用 inode，Windows 使用主控文件表（MFT）
  - 文件控制块（FCB）：目录中每个文件对应一个条目，记录文件的详细信息
- **文件创建流程**
  - 发起 create 系统调用 → 逻辑文件系统分配新 FCB → 从磁盘读取目录到内存 → 写入新条目 → 目录写回磁盘
- **文件打开流程**
  - 发起 open 系统调用 → 查找目录结构定位文件 → 信息拷贝到两级打开文件表
  - 若文件已被其他进程打开，只需在进程打开文件表添加条目并指向系统表
- **文件关闭流程**
  - 删除进程打开文件表对应条目 → 系统表 Open Count 减一
  - Open Count 为零时将文件写回磁盘、更新目录并删除系统表条目

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 引导控制块 | Boot Control Block | 记录操作系统在磁盘位置的数据结构 |
| 卷控制块 | Volume Control Block | 记录卷的块数、大小、空闲状态等信息 |
| 文件控制块 | FCB (File Control Block) | 目录中每个文件对应的信息条目 |
| 元数据 | Meta Data | 文件系统的目录结构及文件属性信息 |
| 逻辑文件系统 | Logical File System | 文件系统最上层，管理所有元数据 |
| inode | inode | Linux/Unix 中保存文件信息的数据结构 |

## 要点总结

1. 文件系统采用**四层分层结构**：I/O控制层、基础文件系统、文件组织模块、逻辑文件系统
2. 磁盘以**块**为基本单位，典型大小为 512B 或 4KB，支持直接访问
3. **引导控制块**定位操作系统，**卷控制块**记录卷信息，**FCB** 记录单个文件信息
4. 文件打开时信息分别存入**两级打开文件表**，已打开的文件可被多进程共享
5. 关闭文件时 **Open Count** 降为零才将文件写回磁盘并清理系统表条目
