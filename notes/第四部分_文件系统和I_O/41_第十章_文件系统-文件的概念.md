# 第十章 文件系统——文件的概念

## 本讲主题

介绍文件的定义、类型、属性、基本操作及两级打开文件表机制。

## 核心知识点

- **文件的定义与作用**
  - 文件是逻辑上统一的信息保存单元，屏蔽底层存储设备的异构性
  - 底层可为磁信号（磁盘）、光信号（光盘）、电信号（网络存储）等
  - 文件由数据（Data）和程序/格式信息（Program）两部分组成
- **文件类型**
  - 操作系统视角：文本文件（Text）、源代码文件（Source）、目标文件（OBJ）、可执行文件（Executable）
  - 用户视角：文本、多媒体、可执行文件等更多分类
- **文件属性**
  - 名称（Name）：人类可读的标识
  - 标识符（Identifier）：操作系统内部的唯一数字标识
  - 类型（Type）：以扩展名形式表示
  - 位置（Location）、大小（Size）、保护（Protection）、时间信息等
  - 文件属性保存在文件系统的目录结构中
- **文件操作**
  - 六种基本操作：创建（Create）、写入（Write）、读取（Read）、重定位（Seek）、删除（Delete）、截断（Truncate）
  - 读写操作各维护一个指针，统称当前文件位置指针（Current File Position Pointer）
  - 截断保留文件属性但删除所有数据
- **打开文件表**
  - 系统范围打开文件表：记录与进程无关的信息（磁盘位置、创建日期、Open Count）
  - 单个进程打开文件表：记录与进程相关的信息（读写指针位置、访问权限）
  - 两级表通过指针关联：进程表条目指向系统表对应条目
- **文件作为抽象数据类型**
  - 文件类似面向对象中的类，同类型文件拥有相同属性和操作，数据不同

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 文件 | File | 逻辑上统一的信息保存单元 |
| 抽象数据类型 | Abstract Data Type | 定义属性和操作的类型模板，可产生多个实例 |
| 文件位置指针 | Current File Position Pointer | 标识文件当前读写位置的指针 |
| 截断 | Truncate | 删除文件数据但保留属性的操作 |
| 系统打开文件表 | System-wide Open File Table | 记录全系统所有已打开文件的表 |
| 进程打开文件表 | Per-process Open File Table | 记录单个进程已打开文件的表 |
| 打开计数 | Open Count | 记录文件被多少个进程打开的计数器 |

## 重要对比/关系

| 对比项 | 系统范围打开文件表 | 单个进程打开文件表 |
|--------|-------------------|-------------------|
| 作用域 | 整个操作系统 | 单个进程 |
| 存储内容 | 磁盘位置、创建日期、Open Count | 读写指针、访问权限 |
| 与进程关系 | 无关 | 相关 |

## 要点总结

1. 文件是屏蔽底层存储异构性的**逻辑保存单元**，由数据和程序/格式信息组成
2. 文件具有名称、标识符、类型等属性，所有属性保存在**目录结构**中
3. 六种基本操作中，**读写操作**各维护独立指针，Seek 用于重定位指针
4. 操作系统维护**两级打开文件表**，分别记录与进程无关和相关的文件信息
5. 关闭文件时 Open Count 减一，降为零则将更新内容**写回磁盘**并删除系统表条目
