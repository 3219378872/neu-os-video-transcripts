# 第十二章 I/O子系统——设备与接口

## 本讲主题

介绍 I/O 设备的多样性与标准化趋势、硬件连接方式、CPU 与控制器的通信及 I/O 访问方式。

## 核心知识点

- **I/O 设备的两种趋势**
  - 设备种类越来越多（键盘、鼠标、VR/AR 设备、游戏外设等）
  - 硬件接口越来越标准化（USB 统一接口趋势）
- **USB 接口类型**
  - Type A：早期 USB 2.0，速度较低
  - Type A Super Speed：USB 3.0，速度更高
  - Type B / Type B Super Speed：用于 3.5 寸移动硬盘
  - Mini A/B、Micro A/B：早期安卓手机充电口
  - Micro B Super Speed：用于 2.5 寸移动硬盘
  - Type C：最新标准，苹果笔记本和新安卓手机广泛采用
- **硬件连接三要素**
  - 端口（Port）：I/O 设备插入计算机的接口
  - 总线（Bus）：传输线路及其上的通信协议（如 PCI 总线、扩展总线）
  - 控制器（Controller）：管理一类同种设备，一个控制器对应多个同类设备
- **CPU 与控制器的通信方式**
  - 直接 I/O 指令：CPU 直接通过总线将命令写入控制器寄存器
  - 内存映射 I/O：将控制器寄存器映射到内存地址范围，CPU 通过正常寻址通信
- **I/O 通信策略**
  - 轮询（Polling）：CPU 主动查询设备忙位（Busy Bit），空闲则访问
  - 中断（Interrupt）：设备控制器主动通知 CPU 任务已完成或设备已空闲
- **DMA 直接内存访问**
  - CPU 发出 DMA 指令写入 DMA 控制器（含磁盘地址、数据量、控制信息）
  - DMA 控制器代替 CPU 管理数据从磁盘到内存缓冲区的传输
  - 传输完成后由 DMA 控制器通过中断通知 CPU
- **I/O 设备分类**
  - 块设备（磁盘）：read/write/seek 系统调用
  - 字符设备（键盘、鼠标、串口）：get/put 指令
  - 时钟与定时器（Clock/Timer）：提供当前时间、计时、定时中断（如时间片轮转）

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 端口 | Port | I/O 设备连接计算机的物理接口 |
| 总线 | Bus | 传输线路及通信协议的集合 |
| 设备控制器 | Controller | 管理一类同种 I/O 设备的硬件 |
| 轮询 | Polling | CPU 主动查询设备状态的通信方式 |
| 中断 | Interrupt | 设备主动通知 CPU 的通信方式 |
| 直接内存访问 | DMA (Direct Memory Access) | 绕过 CPU 直接在设备与内存间传输数据 |

## 重要对比/关系

| 对比项 | 轮询 | 中断 |
|--------|------|------|
| 发起方 | CPU 主动查询 | 设备主动通知 |
| CPU 开销 | 高（反复查询） | 低（仅响应中断） |
| 适用场景 | 高速设备、短等待 | 低速设备、长等待 |

## 要点总结

1. I/O 设备种类日益增多，但接口趋向**标准化**（USB Type C 成为主流）
2. 设备通过**端口、总线、控制器**三层连接到计算机系统
3. CPU 与控制器通信有**直接 I/O 指令**和**内存映射 I/O** 两种方式
4. **DMA** 使 CPU 从大块数据传输中解脱，由 DMA 控制器代为管理传输过程
5. I/O 设备按数据粒度分为**块设备、字符设备和时钟/定时器**三类
