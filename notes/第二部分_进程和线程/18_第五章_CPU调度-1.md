# 第五章 CPU 调度（一）

## 本讲主题

介绍 CPU 调度的基本概念、调度准则（五种评价指标），以及先到先服务（FCFS）和最短作业优先（SJF）两种调度算法。

## 核心知识点

- **CPU 区间与 I/O 区间**
  - 进程运行由 CPU burst 和 I/O burst 交替组成
  - 始于 CPU burst，终于 CPU burst
  - 真实系统中大量短 CPU 区间、少量长 CPU 区间
- **CPU 调度器与分派器**
  - CPU 调度器（Scheduler）：按规则从 Ready Queue 选择进程
  - 分派器（Dispatcher）：将 CPU 使用权交给被选进程（含上下文切换、模式转换）
  - 抢占式（Preemptive）：允许新进程抢占当前运行进程的 CPU
  - 非抢占式（Non-preemptive）：进程获得 CPU 后一直运行到完成或自愿放弃
- **五种调度准则**
  - **CPU 利用率**：应尽可能高
  - **吞吐量（Throughput）**：单位时间完成的进程数
  - **周转时间（Turnaround Time）**：从进入 Ready Queue 到运行完毕的总时间 = 等待时间 + 执行时间
  - **等待时间（Waiting Time）**：在 Ready Queue 中等待的时间（最重要指标之一）
  - **响应时间（Response Time）**：从提交请求到首次响应的时间
- **先到先服务（FCFS）**
  - 按到达顺序以 FIFO 队列组织
  - 平均等待时间受到达顺序影响极大
  - 示例：P1(24ms)、P2(3ms)、P3(3ms) → 顺序 P1P2P3 平均等待 17ms，顺序 P2P3P1 平均等待 3ms
- **最短作业优先（SJF）**
  - 选择 CPU 区间最短的进程优先执行
  - 相同区间长度时使用 FCFS
  - 理论最优，但需预测 CPU 区间长度
  - **指数平均预测**：τ(n+1) = α·t(n) + (1-α)·τ(n)，通过历史数据估计下一个 CPU 区间

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| CPU 区间 | CPU Burst | 进程在 CPU 上连续执行的时间段 |
| 调度器 | CPU Scheduler | 按规则从 Ready Queue 选择进程的组件 |
| 分派器 | Dispatcher | 将 CPU 交给选中进程并启动执行的组件 |
| 周转时间 | Turnaround Time | 进程从进入系统到完成的总时间 |
| 等待时间 | Waiting Time | 进程在 Ready Queue 中等待的时间 |
| 先到先服务 | FCFS | 按到达顺序执行的调度算法 |
| 最短作业优先 | SJF | 选择最短 CPU 区间进程优先执行 |
| 指数平均 | Exponential Averaging | 用历史数据加权预测下一个 CPU 区间长度 |

## 要点总结

1. CPU 调度的核心目标：**最大化利用率和吞吐量，最小化周转时间、等待时间和响应时间**
2. FCFS 最简单但平均等待时间**受到达顺序影响极大**，短作业在长作业后等待时间很长
3. SJF 提供**理论最优**的平均等待时间，但需预测未来 CPU 区间长度
4. 指数平均法 τ(n+1) = α·t(n) + (1-α)·τ(n) 可通过历史数据有效估计 CPU 区间
5. 调度算法分为**抢占式**和**非抢占式**两大类
