# 第六章 进程同步——信号量

## 本讲主题

介绍 Dijkstra 提出的信号量（Semaphore）机制，包括基本定义、改进定义（结构体信号量），以及计数信号量和互斥信号量。

## 核心知识点

- **信号量的基本定义**
  - 由 Dijkstra 于 1965 年提出
  - 信号量 S 本质是一个整数，只能通过两种原子操作访问：
    - **wait(S)**：while(S ≤ 0); S--; （P 操作）
    - **signal(S)**：S++; （V 操作）
  - P/V 源自荷兰语（Dijkstra 是荷兰人）
- **用信号量解决互斥**
  - 定义 mutex 信号量，初始化为 1
  - 进入临界区前执行 wait(mutex)，离开后执行 signal(mutex)
  - 第一个进程进入后 mutex=0，后续进程在 wait 中循环等待
- **用信号量控制执行顺序**
  - 定义信号量 synch 初始化为 0
  - P1 执行 S1 后 signal(synch)；P2 先 wait(synch) 再执行 S2
  - 保证 S2 一定在 S1 之后执行
- **改进的信号量定义（结构体）**
  - 解决忙等待（Busy Waiting）问题
  - 信号量定义为结构体：整数值 value + 等待队列
  - 增加两种进程操作：block（阻塞进程）和 wakeup（唤醒进程）
  - 改进的 wait：S->value--; if(S->value < 0) 将进程加入等待队列并 block
  - 改进的 signal：S->value++; if(S->value ≤ 0) 从等待队列唤醒一个进程
- **信号量的分类**
  - **计数信号量（Counting Semaphore）**：控制有限资源（如 3 台打印机 → S=3）
  - **互斥信号量（Binary Semaphore）**：值只有 0 和 1，用于互斥访问
- **信号量可能导致的问题**
  - **死锁**：两个进程各持有一个信号量并等待对方的信号量
  - **饥饿**：进程一直在等待队列中未被唤醒

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 信号量 | Semaphore | 用于进程同步的整数变量，通过 wait/signal 操作访问 |
| wait (P) | wait / P operation | 信号量减 1，若 ≤0 则等待 |
| signal (V) | signal / V operation | 信号量加 1，若 ≤0 则唤醒等待进程 |
| 计数信号量 | Counting Semaphore | 控制多个有限资源实例的信号量 |
| 互斥信号量 | Binary Semaphore | 值为 0/1，用于互斥访问的信号量 |
| 忙等待 | Busy Waiting | 进程在循环中空转等待，浪费 CPU |

## 要点总结

1. 信号量是最常用的进程同步机制，通过 **wait（P）和 signal（V）**两种原子操作控制
2. 基本定义中 wait 使用忙等待；改进定义用**等待队列 + block/wakeup** 避免浪费 CPU
3. 计数信号量控制有限资源（S=资源数），互斥信号量（S=1）控制互斥访问
4. 信号量使用不当可能导致**死锁**（互相等待）和**饥饿**（永远未被唤醒）
5. wait 和 signal 必须**成对使用**且顺序正确，否则会破坏互斥或产生死锁
