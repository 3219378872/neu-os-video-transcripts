# 第四章 线程（下）

## 本讲主题

介绍用户级线程与内核级线程的分类、各自优缺点、映射模型，以及线程池和 Java 线程。

## 核心知识点

- **用户级线程（User-level Thread）**
  - 通过函数库在用户空间实现，内核不知道线程的存在
  - 应用场景：早期不支持线程的 OS；跨平台需求
  - 调度方式：OS 按进程调度，进程内部自行分配时间片给各线程
  - 缺点：一个线程执行系统调用 → 整个进程进入 Waiting 状态，所有线程阻塞
  - 优点：切换灵活、可设计独特调度算法、与 OS 无关
- **内核级线程（Kernel-level Thread）**
  - 线程的创建、调度、结束全部在内核空间完成
  - OS 内部同时维护 Process Table 和 Thread Table
  - Ready Queue 中等待的是线程而非进程
  - 优点：多处理器上可并行、一个线程阻塞不影响其他线程、真正的线程级调度
  - 缺点：线程操作需从用户态切换到内核态
- **用户级线程到内核级线程的映射**
  - **多对一（Many-to-One）**：多个用户线程映射到一个内核线程（如早期 Solaris）
  - **一对一（One-to-One）**：每个用户线程对应一个内核线程（如 Windows XP）
  - **多对多（Many-to-Many）**：多个用户线程映射到多个内核线程，不保持一一对应（如 Tru64 UNIX）
- **线程池（Thread Pool）**
  - 系统初始化时预先创建一批线程
  - 请求到来时分配已有线程服务，完成后线程归还线程池
  - 避免了每次创建/销毁线程的开销
- **Java 线程**
  - 两种实现方式：继承 Thread 类 / 实现 Runnable 接口
  - 由 JVM 管理，映射到不同 OS 的内核级线程

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 用户级线程 | User-level Thread | 在用户空间通过函数库实现的线程 |
| 内核级线程 | Kernel-level Thread | 由操作系统内核直接管理的线程 |
| 多对一映射 | Many-to-One | 多个用户线程映射到一个内核线程 |
| 一对一映射 | One-to-One | 每个用户线程对应一个内核线程 |
| 多对多映射 | Many-to-Many | 多个用户线程映射到多个内核线程 |
| 线程池 | Thread Pool | 预创建线程集合，按需分配服务 |

## 重要对比/关系

| 对比项 | 用户级线程 | 内核级线程 |
|--------|-----------|-----------|
| 管理者 | 用户空间函数库 | 操作系统内核 |
| OS 可见性 | 不可见 | 可见 |
| 调度单位 | 进程 | 线程 |
| 系统调用影响 | 整个进程阻塞 | 仅该线程阻塞 |
| 切换效率 | 高（无模式切换） | 较低（需模式切换） |

## 要点总结

1. 用户级线程在用户空间管理，**OS 不可见**，一个线程系统调用会阻塞整个进程
2. 内核级线程由 OS 直接管理，**真正实现线程级调度**，一个线程阻塞不影响其他线程
3. 用户线程到内核线程的映射有**多对一、一对一、多对多**三种模型
4. **线程池**通过预创建线程避免频繁创建/销毁的开销，广泛用于 Web 服务器
5. Java 线程由 JVM 管理，根据不同 OS 采用不同的映射模型
