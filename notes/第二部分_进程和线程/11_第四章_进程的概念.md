# 第四章 进程的概念

## 本讲主题

介绍进程的定义、进程与程序的区别、进程的五种状态及状态转换，以及进程控制块（PCB）的组成。

## 核心知识点

- **进程的引入背景**
  - 早期批处理系统一次只运行一个程序，CPU 利用率低
  - 多道程序系统让多个程序并发运行，充分利用 CPU 资源
  - 类比：多病人医院需要病历本记录各自状态，避免混淆
- **进程的定义**
  - 一个具有某项功能的程序在某个数据集上的一次执行过程
  - 进程包含一个关键数据结构——PCB（Process Control Block）
  - 进程是资源分配的基本单位；调度和分派的最小单位是线程
  - 概念由 Saltzer 于 1966 年在 MIT 实现 Multics 时首次提出
- **进程与程序的区别**
  - 程序是被动实体（静态文件，存储在磁盘中）
  - 进程是主动实体（正在运行的程序，具有动态性和并发性）
  - 多个进程可对应同一个程序（如同时打开两个 Word 窗口）
- **进程的五种状态**
  - **New**：正在被创建
  - **Running**：指令正在 CPU 中执行
  - **Waiting**：等待某事件（I/O 完成、系统调用返回等）
  - **Ready**：已获得除 CPU 外的所有资源，等待调度
  - **Terminated**：所有指令执行完毕
- **状态转换关系**
  - Ready → Running：调度和分派
  - Running → Terminated：最后一条指令执行完毕
  - Running → Waiting：主动放弃 CPU（申请 I/O 资源）
  - Waiting → Ready：I/O 事件完成
  - Running → Ready：被动放弃 CPU（中断发生）
- **PCB 的七大数据项**
  - 进程状态（Process State）
  - 程序计数器（Program Counter）
  - CPU 寄存器（Registers）
  - CPU 调度信息（优先级、调度队列等）
  - 内存管理信息（基地址寄存器、界限寄存器等）
  - 记账信息（CPU 使用时间、时间界限等）
  - I/O 状态信息（已分配的设备、文件等）

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 进程 | Process | 程序在某数据集上的一次执行过程 |
| 程序 | Program | 存储在磁盘中的被动实体（静态文件） |
| 进程控制块 | Process Control Block (PCB) | 记录进程所有执行状态和运行环境的数据结构 |
| 程序计数器 | Program Counter (PC) | 指向当前进程下一条将要执行指令的内存地址 |
| 寄存器 | Register | 保存进程当前执行状态的 CPU 硬件 |
| 多道程序系统 | Multiprogramming System | 允许多个程序并发运行的计算机系统 |

## 重要对比/关系

| 对比项 | 程序 | 进程 |
|--------|------|------|
| 性质 | 被动实体 | 主动实体 |
| 存储 | 磁盘中的文件 | 内存中运行 |
| 动态性 | 静态 | 动态（PC 高速变化） |
| 对应关系 | 一个程序 | 可对应多个进程 |

## 要点总结

1. 进程是程序在数据集上的一次执行过程，包含关键数据结构 **PCB**
2. 程序是被动的静态文件，进程是主动的动态执行，二者可存在**一对多**关系
3. 进程具有 **New、Running、Waiting、Ready、Terminated** 五种状态
4. Running → Waiting 是**主动**放弃 CPU，Running → Ready 是**被动**放弃 CPU
5. PCB 包含进程状态、PC、寄存器、调度信息、内存信息、记账信息和 I/O 信息七大数据项
