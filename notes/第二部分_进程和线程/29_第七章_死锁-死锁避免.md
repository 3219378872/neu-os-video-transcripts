# 第七章 死锁——死锁避免

## 本讲主题

介绍死锁避免（Deadlock Avoidance）策略，包括安全状态与安全序列的概念，以及银行家算法。

## 核心知识点

- **死锁避免的基本思想**
  - 每个进程入系统时声明各类资源的最大需求量
  - 资源分配前动态检测：分配后系统是否仍安全
  - 若分配导致不安全状态 → 不分配，进程等待
- **安全状态与安全序列**
  - 安全状态：能找到一个进程执行顺序（安全序列），使每个进程都能获得资源并完成
  - 安全序列不唯一
  - 示例：12 个磁带，P0 需 10（已分 5）、P1 需 4（已分 2）、P2 需 9（已分 2），剩余 3 → 安全序列 P1→P0→P2
- **资源分配图方法（每类资源一个实例）**
  - 增加**需求边**（Claim Edge，虚线）：进程未来可能申请的资源
  - 申请时虚线变实线（申请边），分配后变分配边，释放后恢复虚线
  - 判断：申请边转为分配边后，若包含需求边的图形成环 → 可能死锁 → 不分配
- **银行家算法（每类资源多个实例）**
  - 数据结构（n 个进程，m 类资源）：
    - **Available**[m]：各类资源剩余实例数
    - **Max**[n][m]：每个进程对每类资源的最大需求
    - **Allocation**[n][m]：已分配给每个进程的资源
    - **Need**[n][m]：还需要的资源（Need = Max - Allocation）
  - **安全性算法**：
    1. Work = Available, Finish[i] = false
    2. 找 Finish[i]=false 且 Need[i] ≤ Work 的进程 i
    3. Work = Work + Allocation[i], Finish[i] = true
    4. 重复直到所有 Finish 为 true（安全）或找不到满足条件的进程（不安全）
  - **资源请求算法**：
    1. 检查 Request[i] ≤ Need[i]
    2. 检查 Request[i] ≤ Available
    3. 假设分配，更新数据结构
    4. 运行安全性算法：安全 → 实际分配；不安全 → 回滚，进程等待

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 死锁避免 | Deadlock Avoidance | 动态检测资源分配是否导致不安全状态 |
| 安全状态 | Safe State | 能找到安全序列使所有进程完成 |
| 安全序列 | Safe Sequence | 进程的一种执行顺序，保证每个都能获得资源 |
| 银行家算法 | Banker's Algorithm | 通过安全性检测决定是否分配资源的算法 |
| 需求边 | Claim Edge | 资源分配图中表示未来可能申请的虚线边 |

## 要点总结

1. 死锁避免在**每次资源分配前**动态检测系统安全性，而非静态破坏必要条件
2. 系统安全的标志是能找到一个**安全序列**，使所有进程都能按序获得资源并完成
3. 银行家算法包含**安全性算法**和**资源请求算法**两部分
4. 资源请求时先**假设分配**再检测安全性，不安全则回滚并让进程等待
5. Need = Max - Allocation 是银行家算法的核心关系式
