# 第七章 死锁——检测与恢复

## 本讲主题

通过银行家算法实例演示安全性检测过程，介绍死锁检测算法（等待图和类银行家算法）和死锁恢复策略。

## 核心知识点

- **银行家算法实例**
  - 5 个进程（P0-P4），3 类资源 A(10)、B(5)、C(7)
  - Available = [3,3,2]，Need = Max - Allocation
  - 安全性检测过程：P1(可执行) → P3(可执行) → P4 → P0 → P2
  - 安全序列：P1, P3, P4, P0, P2 → 系统安全
  - P1 请求 [1,0,2]：Request ≤ Need 且 ≤ Available → 假设分配 → 安全 → 实际分配
- **死锁检测算法**
  - **每类资源一个实例 → 等待图（Wait-for Graph）**
    - 从资源分配图删除资源节点，保留进程间的等待关系
    - P1→R1→P2 变为 P1→P2（P1 等待 P2 释放资源）
    - 等待图有环 → 死锁；无环 → 无死锁
  - **每类资源多个实例 → 类银行家算法**
    - 数据结构：Available、Allocation、Request（替代 Need）
    - 初始化：Allocation[i]=0 的进程 Finish[i]=true（不持有资源则不会死锁）
    - 找 Finish[i]=false 且 Request[i] ≤ Work 的进程执行
    - 所有 Finish 为 true → 无死锁；存在 false → 对应进程处于死锁
  - 检测实例：P2 请求 [0,0,1] 后，只有 P0 可执行 → P1,P2,P3,P4 死锁
- **死锁恢复策略**
  - **进程终止法**：
    - 全部终止：停止所有死锁进程，回滚到执行前状态
    - 逐个终止：每次停一个，检测是否解除死锁，释放该进程资源
  - **资源抢占法**：
    - 允许一个进程抢占另一个死锁进程的资源

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 等待图 | Wait-for Graph | 删除资源节点后的进程等待关系图 |
| 死锁检测 | Deadlock Detection | 检测系统当前是否处于死锁状态的算法 |
| 死锁恢复 | Deadlock Recovery | 检测到死锁后使系统恢复正常的策略 |
| 进程终止 | Process Termination | 停止死锁进程并释放资源 |
| 资源抢占 | Resource Preemption | 强制从一个进程夺取资源给另一个进程 |

## 重要对比/关系

| 策略 | 思路 | 代价 |
|------|------|------|
| 死锁预防 | 破坏必要条件 | 资源利用率低 |
| 死锁避免 | 动态安全性检测 | 每次分配需运行算法 |
| 死锁检测+恢复 | 允许死锁后处理 | 需回滚/终止进程 |
| 忽略 | 假设不发生 | 需重启系统 |

## 要点总结

1. 银行家算法通过**假设分配 → 安全性检测 → 确认/回滚**的流程决定资源分配
2. 等待图是资源分配图的简化，**有环则死锁**，适用于每类资源一个实例的情况
3. 类银行家检测算法中，**不持有资源的进程**（Allocation=0）直接标记为 Finish=true
4. 死锁恢复有**进程终止**（全部/逐个）和**资源抢占**两种策略
5. 四种死锁处理策略（预防、避免、检测恢复、忽略）各有**代价与适用场景**，需权衡选择
