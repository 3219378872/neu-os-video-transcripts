# 第四章 进程操作

## 本讲主题

介绍进程的创建过程（树状组织结构、四步创建流程）、UNIX 中的 fork/exec/wait 系统调用，以及进程终止的情形。

## 核心知识点

- **进程的树状组织**
  - 操作系统中所有进程以树状结构组织
  - 父进程创建子进程，子进程可进一步创建子进程
  - 每个进程有唯一的 PID（Process Identifier）
  - Solaris 示例：根进程 SCHED（PID=0）→ init（PID=1, 用户进程根）、PageOut（PID=2, 内存管理）、fsflush（PID=3, 文件管理）
- **父子进程的资源关系**
  - 共享所有资源 / 共享资源子集 / 不共享资源（向 OS 申请）
- **父子进程的执行方式**
  - 并发执行：父子进程同时在 Ready Queue 中
  - 父进程等待：父进程 block，让子进程先执行
- **进程创建的四个步骤**
  1. 分配空白 PCB
  2. 分配所需系统资源（内存、I/O、文件等）
  3. 初始化 PCB（设置 PC 指向第一条指令、调度参数等）
  4. 将进程放入 Ready Queue 参与调度
- **UNIX 关键系统调用**
  - **fork()**：创建子进程，子进程拥有与父进程相同的地址空间
    - 父进程返回值：子进程的 PID（>0）
    - 子进程返回值：0
    - 通过返回值区分父子进程，避免无限递归创建
  - **exec()**：将新程序加载到子进程的地址空间中执行
  - **wait()**：父进程进入等待状态，直到子进程执行完毕后被唤醒
  - **exit()**：子进程通知 OS 执行完毕
- **进程终止**
  - 正常终止：最后一条指令执行完毕，资源被 OS 回收
  - 异常终止：子进程越权访问资源、父进程认为子进程不再需要

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 进程标识符 | Process Identifier (PID) | 操作系统为每个进程分配的唯一整数标识 |
| fork | fork() | UNIX 系统调用，创建子进程并复制父进程地址空间 |
| exec | exec() | 将新程序加载到当前进程地址空间并执行 |
| wait | wait() | 父进程等待子进程完成执行 |
| exit | exit() | 进程通知操作系统自己已执行完毕 |
| POSIX | POSIX | 可移植操作系统接口标准 |

## 要点总结

1. 操作系统中所有进程以**树状结构**组织，每个进程有唯一 **PID**
2. 进程创建经历**分配 PCB → 分配资源 → 初始化 PCB → 进入 Ready Queue** 四步
3. fork() 的关键设计：父进程返回子进程 PID（>0），子进程返回 0，通过返回值区分角色
4. fork() + exec() + wait() 三者协作完成**创建子进程 → 加载新程序 → 等待完成**的完整流程
5. 进程终止分为正常执行完毕和异常终止（越权、不再需要），资源均由 OS 回收
