# 第六章 进程同步——硬件同步

## 本讲主题

介绍利用硬件指令 TestAndSet 和 Swap 解决临界区问题的方法，强调原子操作的重要性。

## 核心知识点

- **硬件同步的前提**
  - 硬件指令内部的语句必须以**原子方式**执行（要么全部执行，要么都不执行）
  - 执行过程中不允许产生中断
- **TestAndSet 指令**
  - 定义：`bool TestAndSet(bool *target) { bool rv = *target; *target = true; return rv; }`
  - 行为：返回值等于输入值；执行后输入参数**一定变为 true**
  - 使用方式：
    - 共享变量 `lock` 初始化为 false
    - `while(TestAndSet(&lock));`（进入区）
    - 临界区代码
    - `lock = false;`（离开区）
  - 工作原理：第一个进程执行时 lock=false → 返回 false → 跳出循环进入临界区，同时 lock 变为 true；后续进程执行时 lock=true → 返回 true → 空循环等待
- **Swap 指令**
  - 定义：`void Swap(bool *a, bool *b) { bool temp = *a; *a = *b; *b = temp; }`
  - 交换两个布尔变量的值
  - 使用方式：
    - 共享变量 `lock` 初始化为 false
    - 每个进程有本地变量 `key`
    - `key = true; while(key == true) Swap(&lock, &key);`（进入区）
    - 临界区代码
    - `lock = false;`（离开区）
  - 工作原理：lock=false 时 Swap 后 key=false → 跳出循环；lock=true 时 Swap 后 key 仍为 true → 继续循环

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 原子操作 | Atomic Operation | 执行过程中不可被中断的操作 |
| TestAndSet | TestAndSet | 返回原值并将变量设为 true 的原子指令 |
| Swap | Swap | 原子交换两个变量值的硬件指令 |
| 忙等待 | Busy Waiting | 进程在循环中不断检测条件（浪费 CPU） |

## 重要对比/关系

| 对比项 | TestAndSet | Swap |
|--------|-----------|------|
| 返回值 | 有（bool） | 无（void） |
| 变量数 | 1 个共享 lock | 1 个共享 lock + 每进程 1 个 key |
| 原理 | 返回旧值，设为 true | 交换 lock 和 key 的值 |

## 要点总结

1. 硬件同步的关键是指令必须以**原子方式**执行，不允许中间被中断
2. TestAndSet 的核心：返回值等于输入值，但执行后输入参数**一定变为 true**
3. Swap 通过交换共享 lock 和本地 key 的值来判断是否可进入临界区
4. 两种硬件指令都能满足**互斥**条件，通过离开区将 lock 设为 false 释放锁
5. 硬件同步方案的缺点是**忙等待**（Busy Waiting），浪费 CPU 时钟周期
