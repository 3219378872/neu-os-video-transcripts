# 第五章 CPU 调度（三）

## 本讲主题

介绍多级队列调度和多级反馈队列调度两种策略性调度方法，分析真实系统中的调度方案。

## 核心知识点

- **多级队列调度（Multilevel Queue Scheduling）**
  - 将 Ready Queue 分成多个子队列，按进程性质永久分配
  - 不同队列可采用不同调度算法
    - 交互式进程队列：Round Robin（最好的交互性能）
    - 批处理/CPU 密集型队列：FCFS（给予大段连续 CPU 时间）
  - 队列间调度的两种策略：
    - **固定优先级**：高优先级队列不空时，低优先级队列不执行（抢占式）
    - **时间片分配**：不同队列获得不同比例的 CPU 时间（如交互式 80%，批处理 20%）
  - 典型五级队列：系统进程 > 交互式进程 > 交互式文本编辑 > 批处理进程 > 学生实验进程
- **多级反馈队列调度（Multilevel Feedback Queue Scheduling）**
  - 允许进程在不同队列间**动态跳转**
  - 升级：长时间未获 CPU 的进程移到更高优先级队列（防止饥饿）
  - 降级：占用过多 CPU 时间的进程移到更低优先级队列
  - 典型三队列示例：
    - 第一队列：RR，时间片 8ms → 未完成则降到第二队列
    - 第二队列：RR，时间片 16ms → 未完成则降到第三队列
    - 第三队列：FCFS（CPU 密集型进程）
  - 设计考量：队列数量、每队列的调度算法、升级/降级策略、进程初始队列

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 多级队列调度 | Multilevel Queue Scheduling | 将 Ready Queue 分成多个固定子队列的调度 |
| 多级反馈队列调度 | Multilevel Feedback Queue Scheduling | 允许进程在队列间动态迁移的调度 |
| 固定优先级 | Fixed Priority | 高优先级队列空后才执行低优先级队列 |
| 时间片分配 | Time Slice Allocation | 不同队列获得不同比例的 CPU 时间 |
| 交互式进程 | Interactive Process | 需要用户参与的进程 |
| 批处理进程 | Batch Process | CPU 密集型、不需交互的进程 |

## 重要对比/关系

| 对比项 | 多级队列 | 多级反馈队列 |
|--------|---------|-------------|
| 进程跳转 | 不允许 | 允许升级/降级 |
| 灵活性 | 低 | 高 |
| 饥饿风险 | 固定优先级下存在 | 通过升级机制缓解 |
| 适用场景 | 进程性质明确 | 通用系统 |

## 要点总结

1. 多级队列调度按进程性质将 Ready Queue 分成子队列，各队列采用**不同调度算法**
2. 队列间调度有**固定优先级**和**时间片分配**两种策略
3. 多级反馈队列允许进程**动态迁移**，占用 CPU 过多则降级，等待过久则升级
4. 典型设计：短作业在高优先级队列快速完成，长作业逐步降入低优先级队列用 FCFS 调度
5. 多级反馈队列调度是**最通用**的调度方案，大多数现代 OS 采用类似策略
