# 第七章 死锁——必要条件与资源分配图

## 本讲主题

介绍死锁的四个必要条件，以及用资源分配图分析系统是否处于死锁状态。

## 核心知识点

- **死锁的四个必要条件**
  - **互斥（Mutual Exclusion）**：资源一次只能被一个进程使用
  - **占有并等待（Hold and Wait）**：进程持有资源的同时申请新资源
  - **非抢占（No Preemption）**：已分配的资源不能被强制剥夺
  - **循环等待（Circular Wait）**：进程间形成环形等待链（最重要）
  - 四个条件**同时满足**才会发生死锁
- **资源分配图（Resource Allocation Graph）**
  - 两类节点：
    - 进程节点（圆形）
    - 资源节点（方形，内含点表示实例数）
  - 两类边：
    - 申请边（Request Edge）：进程 → 资源（正在申请）
    - 分配边（Assignment Edge）：资源实例 → 进程（已分配）
  - 申请获批后，申请边转为分配边；资源释放后，分配边删除
- **通过资源分配图判断死锁**
  - **无环 → 无死锁**
  - **有环 + 每种资源仅一个实例 → 一定有死锁**
  - **有环 + 资源有多个实例 → 可能有死锁**
  - 示例1：P3 执行完释放 R3 → P2 可执行 → P1 可执行 → 无死锁
  - 示例2：增加 P3→R2 申请边后形成环 → 所有进程无法执行 → 死锁
  - 示例3：4 个进程 2 类资源各 2 实例，有环但 P2、P4 可执行 → 无死锁

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 互斥 | Mutual Exclusion | 资源一次只能被一个进程使用 |
| 占有并等待 | Hold and Wait | 持有资源的同时申请新资源 |
| 非抢占 | No Preemption | 已分配资源不可被强制剥夺 |
| 循环等待 | Circular Wait | 进程间形成环形资源等待链 |
| 资源分配图 | Resource Allocation Graph | 描述进程与资源关系的有向图 |
| 申请边 | Request Edge | 进程指向资源的边 |
| 分配边 | Assignment Edge | 资源实例指向进程的边 |

## 要点总结

1. 死锁四个必要条件：**互斥、占有并等待、非抢占、循环等待**，缺一不可
2. 循环等待是四个条件中**最关键**的，也是分析死锁的核心
3. 资源分配图中**无环则无死锁**，有环需根据资源实例数进一步分析
4. 每种资源仅一个实例时，有环**一定**死锁；多个实例时，有环**可能**死锁
5. 资源分配图是分析死锁的有效**图形化工具**
