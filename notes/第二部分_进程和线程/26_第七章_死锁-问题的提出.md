# 第七章 死锁——问题的提出

## 本讲主题

介绍死锁的定义、系统模型（资源类型与实例），通过独木桥和 DVD 刻录机等示例说明死锁现象。

## 核心知识点

- **死锁的定义**
  - 一组进程中，每个进程都在等待一个事件的发生，而该事件只能由这组进程中的另一个进程引起
  - 所有进程都无法继续执行，形成循环等待
- **死锁产生的背景**
  - 多道程序系统中，多个并发进程竞争有限的系统资源
  - 一个进程使用某些资源的同时申请新资源，而新资源被其他进程占有
  - 形成链式等待时，整条链上的进程都无法执行
- **系统模型**
  - M 种资源类型 R1, R2, ..., RM
  - 每种资源有 Wi 个实例（如 2 块磁盘 → W=2）
  - 进程使用资源的三个步骤：**申请 → 使用 → 释放**
- **死锁示例**
  - 独木桥：两辆车各占一端，互不相让
  - DVD 刻录机：3 个进程各持有 1 台，都需要 2 台，共 3 台 → 死锁
  - 信号量：P0 持有 S 等待 Q，P1 持有 Q 等待 S → 死锁

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 死锁 | Deadlock | 一组进程互相等待对方释放资源，导致全部无法执行 |
| 资源类型 | Resource Type | 系统中某一类资源（如磁盘、打印机） |
| 资源实例 | Resource Instance | 某类资源的具体数量 |
| 循环等待 | Circular Wait | 进程间形成环形的资源等待链 |

## 要点总结

1. 死锁是多个进程**互相等待**对方释放资源，导致所有进程都无法继续执行的状态
2. 资源使用遵循**申请 → 使用 → 释放**三步，使用完毕必须归还系统
3. 死锁不仅出现在系统资源中，**信号量**使用不当也会导致死锁
4. 解决独木桥死锁的方法——让一方先退让——也是计算机中解决死锁的基本思路
5. 死锁问题的研究需要从**必要条件、检测、预防、避免**多角度展开
