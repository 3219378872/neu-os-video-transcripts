# 第五章 CPU 调度（二）

## 本讲主题

介绍抢占式 SJF（最短剩余时间优先）、优先级调度和时间片轮转法（Round Robin），并分析各自特点。

## 核心知识点

- **抢占式 SJF（SRTF）**
  - 也叫最短剩余时间优先（Shortest Remaining Time First）
  - 新进程到达时，与当前进程**剩余时间**比较，更短者抢占 CPU
  - 非抢占式 SJF：当前进程运行完毕后再从 Ready Queue 选最短的
  - 示例（非抢占）：P1(7,0ms)、P2(4,2ms)、P3(1,4ms)、P4(4,5ms) → 平均等待 4ms
  - 示例（抢占）：同上 → 平均等待 3ms
- **优先级调度（Priority Scheduling）**
  - 每个进程附加一个优先级整数，整数越小优先级越高
  - SJF 是优先级调度的特例（以 CPU 区间长度为优先级）
  - 分为抢占式和非抢占式
  - **饥饿问题（Starvation）**：低优先级进程可能永远得不到 CPU
  - **老化（Aging）**：随时间增加低优先级进程的优先级，解决饥饿
- **时间片轮转法（Round Robin, RR）**
  - 专为分时系统设计
  - 定义固定时间片（10-100ms），每个进程运行一个时间片后进入 Ready Queue 队尾
  - Ready Queue 按 FIFO 组织，需要时钟硬件支持
  - 进程区间 < 时间片：主动放弃 CPU
  - 进程区间 > 时间片：时钟中断触发上下文切换
  - n 个进程时，每个进程最多等待 (n-1)×q 时间
  - 时间片太长 → 退化为 FCFS；时间片太短 → 上下文切换过多
  - 周转时间一般比 SJF 长，但**响应时间更好**

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 最短剩余时间优先 | SRTF | 抢占式 SJF，比较新进程与当前进程的剩余时间 |
| 优先级调度 | Priority Scheduling | 按优先级整数选择进程执行 |
| 饥饿 | Starvation | 低优先级进程长期得不到 CPU |
| 老化 | Aging | 逐渐提高等待进程优先级以防饥饿 |
| 时间片轮转 | Round Robin (RR) | 每个进程运行固定时间片后轮转 |
| 时间片 | Time Quantum | RR 中每个进程获得的固定 CPU 时间 |
| 甘特图 | Gantt Chart | 可视化进程调度时间线的图表 |

## 重要对比/关系

| 算法 | 类型 | 最优等待时间 | 响应时间 | 特殊问题 |
|------|------|------------|---------|---------|
| FCFS | 非抢占 | 差 | 差 | 护航效应 |
| SJF | 可选 | 最优 | 一般 | 需预测区间 |
| Priority | 可选 | 依赖优先级 | 一般 | 饥饿 |
| RR | 抢占 | 一般 | 最好 | 时间片选择 |

## 要点总结

1. 抢占式 SJF 比较新进程的 CPU 区间与当前进程的**剩余时间**，更短者抢占
2. 优先级调度可能导致**饥饿**，通过**老化（Aging）**机制逐步提升等待进程的优先级
3. Round Robin 为每个进程分配固定**时间片**，提供最好的**响应时间**，适合分时系统
4. 时间片选择至关重要：太长退化为 FCFS，太短导致上下文切换开销过大
5. 计算等待时间时需注意进程的**到达时间**，而非简单从零时刻计算
