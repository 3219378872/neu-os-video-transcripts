# 第四章 协作进程与生产者消费者

## 本讲主题

介绍独立进程与协作进程的概念、进程间通信的两种方式（共享内存与消息传递），以及有限缓冲区（生产者-消费者）问题。

## 核心知识点

- **独立进程与协作进程**
  - **独立进程**：执行过程中既不影响其他进程，也不受其他进程影响
  - **协作进程**：多个进程彼此协作完成一项任务，互相影响
  - 协作进程的优点：方便模块化编程
  - Chrome 浏览器示例：Browser Process（用户界面/网络连接）+ Renderer Process（每个标签页）+ Plugin Process（每个插件）
- **进程间通信的两种方式**
  - **共享内存（Shared Memory）**：创建时需一次系统调用，之后在内存空间直接交互，无需额外系统调用
  - **消息传递（Message Passing）**：每次通信都需系统调用，通过 OS 维护的消息队列交互
- **有限缓冲区问题（生产者-消费者问题）**
  - 缓冲区类型：无限缓冲区（Unbounded Buffer）vs 有限缓冲区（Bounded Buffer）
  - 两个约束条件：
    - 缓冲区**满**时，生产者必须等待
    - 缓冲区**空**时，消费者必须等待
  - 数据结构定义：
    - `BUFFER_SIZE`：缓冲区大小
    - `in` 指针：指向第一个空位
    - `out` 指针：指向第一个满位
  - 空/满判断：
    - `in == out` → 缓冲区为空
    - `(in + 1) % BUFFER_SIZE == out` → 缓冲区为满
  - 限制：只能有效利用 BUFFER_SIZE - 1 个数据项
- **生产者进程逻辑**：缓冲区满则空循环等待；不满则写入数据并更新 in 指针
- **消费者进程逻辑**：缓冲区空则空循环等待；不空则读取数据并更新 out 指针

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 独立进程 | Independent Process | 不影响也不被其他进程影响的进程 |
| 协作进程 | Cooperating Process | 与其他进程互相影响、共同完成任务的进程 |
| 共享内存 | Shared Memory | 多个进程共享同一块内存区域进行通信 |
| 消息传递 | Message Passing | 通过 OS 维护的消息队列进行进程间通信 |
| 有限缓冲区 | Bounded Buffer | 大小有限制的共享缓冲区 |
| 生产者-消费者 | Producer-Consumer | 经典同步问题，一方写数据一方读数据 |

## 要点总结

1. 协作进程通过**共享内存**或**消息传递**两种方式进行通信
2. 共享内存方式创建时需一次系统调用，后续通信在用户空间完成，效率较高
3. 有限缓冲区问题的两个核心约束：缓冲区满则生产者等待，缓冲区空则消费者等待
4. 通过 `in` 和 `out` 指针配合取模运算实现循环数组，但只能利用 **N-1** 个数据项
5. 生产者-消费者问题是操作系统进程同步的**基础模型**，后续用信号量等机制进一步解决
