# 第六章 进程同步——经典同步问题

## 本讲主题

用信号量解决三个经典同步问题：生产者-消费者问题、读者-写者问题和哲学家进餐问题。

## 核心知识点

- **生产者-消费者问题**
  - 三个信号量：
    - `mutex`（初始 1）：互斥访问共享缓冲区
    - `full`（初始 0）：满项个数
    - `empty`（初始 N）：空项个数
  - 生产者：wait(empty) → wait(mutex) → 写数据 → signal(mutex) → signal(full)
  - 消费者：wait(full) → wait(mutex) → 读数据 → signal(mutex) → signal(empty)
  - empty 控制"缓冲区满时生产者等待"，full 控制"缓冲区空时消费者等待"
- **读者-写者问题（第一读者-写者/读者优先）**
  - 三个条件：写者在数据库中 → 读者等待；无写者 → 读者不等待；读者在数据库中 → 写者等待，其他读者不等待
  - 数据结构：
    - `read_count`（整数，初始 0）：当前读者数
    - `mutex`（信号量，初始 1）：保护 read_count 的互斥访问
    - `wrt`（信号量，初始 1）：读者与写者间的互斥
  - 写者：wait(wrt) → 写操作 → signal(wrt)
  - 读者：wait(mutex) → read_count++ → if(read_count==1) wait(wrt) → signal(mutex) → 读操作 → wait(mutex) → read_count-- → if(read_count==0) signal(wrt) → signal(mutex)
  - 第一个读者获取 wrt 锁阻止写者，最后一个读者释放 wrt 锁
- **哲学家进餐问题**
  - 5 个哲学家围圆桌，5 根筷子，每人需左右两根筷子才能吃饭
  - 每根筷子设一个信号量（初始 1）
  - 基本方案：wait(chopstick[i]) → wait(chopstick[(i+1)%5]) → eat → signal 两根
  - 死锁风险：5 人同时拿起左手筷子 → 全部等待右手筷子
  - 解决方案：
    - 最多允许 4 人同时拿筷子
    - 两根筷子都可用时才允许拿起
    - 奇数号先拿左手，偶数号先拿右手
- **信号量使用注意**
  - wait 和 signal 顺序写反 → 破坏互斥
  - 两个 wait 无 signal → 死锁
  - 忘写 wait 或 signal → 错误行为

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 生产者-消费者 | Producer-Consumer | 一方写一方读共享缓冲区的同步问题 |
| 读者-写者 | Readers-Writers | 多读者可并发读但写者需互斥的同步问题 |
| 哲学家进餐 | Dining Philosophers | 资源竞争的经典同步问题 |
| 读者优先 | First Readers-Writers | 读者在场时写者等待的策略 |

## 要点总结

1. 生产者-消费者用三个信号量（**mutex、full、empty**）配合四条语句完整解决
2. 读者-写者问题的关键：第一个读者获取 wrt 锁，最后一个读者释放 wrt 锁
3. 哲学家进餐的基本方案有**死锁风险**，需通过限制并发数等策略解决
4. 信号量的 wait 和 signal 必须**正确配对、正确排序**，否则导致互斥失败或死锁
5. 三个经典问题覆盖了进程同步的典型场景：**互斥访问、条件等待、资源竞争**
