# 第四章 进程调度

## 本讲主题

介绍进程调度的基本数据结构（作业队列、就绪队列、设备队列）、长期/短期/中期调度的概念，以及上下文切换。

## 核心知识点

- **三种基本队列**
  - **作业队列（Job Queue）**：系统中所有已提交的进程
  - **就绪队列（Ready Queue）**：已获得除 CPU 外所有资源、等待执行的进程
  - **设备队列（Device Queue）**：每个 I/O 设备对应一个等待队列
  - 队列中保存的是进程的 PCB，可用链表、线性表或索引表组织
- **队列图（Queueing Diagram）**
  - 长方形块表示队列，圆形表示资源
  - 进程在不同队列间迁移，完成执行过程
  - 从 CPU 出来的分支：终止、I/O 请求、时间片到期、Fork 子进程、等待中断
- **三级调度**
  - **长期调度（作业调度）**：从作业队列选择子集加载到 Ready Queue
    - 决定多道程序的度
    - 执行频率低（秒/分钟级），可精心设计
    - 决定进程组合（I/O-bound 与 CPU-bound 的平衡）
  - **短期调度（CPU 调度）**：从 Ready Queue 选择进程加载到 CPU
    - 执行频率高（毫秒级），要求快速
  - **中期调度**：将进程从内存移出到作业池
    - 改善进程组合
    - 减小多道程序的度，防止系统颠簸
- **进程组合（Process Mix）**
  - I/O 密集型进程（I/O-bound）：大量时间花在 I/O 上
  - CPU 密集型进程（CPU-bound）：大量时间花在 CPU 计算上
  - 好的进程组合应兼具两类进程，避免 CPU 或 I/O 设备空闲
- **上下文切换（Context Switch）**
  - 保存当前进程的运行状态到其 PCB
  - 加载将要执行进程的 PCB 中的状态
  - 上下文切换本身有时间开销

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 作业队列 | Job Queue | 系统中所有已提交进程的集合 |
| 就绪队列 | Ready Queue | 等待 CPU 调度的进程集合 |
| 设备队列 | Device Queue | 等待某 I/O 设备的进程队列 |
| 长期调度 | Long-term Scheduling | 从作业池选进程加载到内存 |
| 短期调度 | Short-term Scheduling | 从 Ready Queue 选进程到 CPU |
| 中期调度 | Medium-term Scheduling | 将进程从内存移出以调整度 |
| 上下文切换 | Context Switch | CPU 切换进程时保存/加载状态的过程 |
| I/O 密集型 | I/O-bound Process | 花费更多时间在 I/O 操作的进程 |
| CPU 密集型 | CPU-bound Process | 花费更多时间在 CPU 计算的进程 |

## 要点总结

1. 进程在执行过程中在 **Ready Queue、Device Queue** 等不同队列间迁移
2. 长期调度决定**多道程序的度**和**进程组合**，短期调度要求**快速**执行
3. 好的进程组合应平衡 **I/O-bound** 和 **CPU-bound** 进程，避免资源空闲
4. 中期调度通过将进程移出内存来**改善进程组合**和**减小多道程序的度**
5. 上下文切换需保存/加载 PCB 状态，其**开销**直接影响系统效率
