# 第四章 线程（上）

## 本讲主题

介绍线程的提出背景、定义，以及进程与线程的区别与联系。

## 核心知识点

- **线程的提出背景**
  - 20 世纪 80 年代后期提出
  - 进程的上下文切换和创建开销大，影响多道程序执行效率
  - 需要更轻量级的并发数据结构
- **进程属性的两类分离**
  - **资源相关属性**：内存地址空间、数据段、代码段、文件、I/O 资源
  - **执行相关属性**：Program Counter、Register、Stack
  - 将执行相关属性抽象为**线程**，实现更高效的并发
- **线程的定义**
  - 线程是 CPU 运行的基本单元
  - 包含：线程 ID、Program Counter、Register、Stack
  - 同一进程的多个线程共享代码段、数据段和系统资源
  - 线程有自己的 TCB（Thread Control Block）
- **多线程 Web 服务器示例**
  - 多进程方式：每个请求创建一个进程，但资源相同却需重复保存/切换资源信息
  - 多线程方式：一个进程 + 多个线程，线程共享资源，切换时只需保存 PC/Register/Stack
- **进程与线程的区别（面试要点）**
  - 从四个角度回答：**调度、并发、资源、系统开销**
  - 进程是资源分配的基本单位
  - 线程是调度和分派的基本单位
  - 线程不能拥有资源，通过所属进程向 OS 申请
  - 线程切换开销远小于进程切换
- **调度与分派的区别**
  - **调度（Scheduling）**：按规则从竞争者中选择一个
  - **分派（Dispatching）**：将选中的线程加载到 CPU 并运行
- **多线程的好处**
  - 响应速度快
  - 资源共享（如 P2P 下载上传程序）

## 关键术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 线程 | Thread | CPU 执行的基本单元，包含 ID、PC、Register、Stack |
| 线程控制块 | Thread Control Block (TCB) | 保存线程状态信息的数据结构 |
| 调度 | Scheduling | 按规则从多个竞争者中选择一个 |
| 分派 | Dispatching | 将选中的线程/进程加载到 CPU 执行 |

## 重要对比/关系

| 对比项 | 进程 | 线程 |
|--------|------|------|
| 角色 | 资源分配的基本单位 | 调度和分派的基本单位 |
| 资源 | 拥有独立资源 | 共享所属进程的资源 |
| 切换开销 | 大（保存/加载全部 PCB） | 小（仅保存 PC/Register/Stack） |
| 数据结构 | PCB | TCB |

## 要点总结

1. 线程将进程的**执行属性**（PC、Register、Stack）从**资源属性**中分离出来
2. 线程是 **CPU 调度和分派的基本单位**，进程是**资源分配的基本单位**
3. 同一进程的线程**共享**代码段、数据段和系统资源，切换开销远小于进程
4. 面试回答进程与线程区别需从**调度、并发、资源、系统开销**四个角度阐述
5. 多线程优势：**快速响应**和**资源共享**
